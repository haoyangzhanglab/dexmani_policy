# 项目概览

dexmani_policy 是一个灵巧手操作模仿学习框架，基于 Hydra 配置驱动、Diffusion/FlowMatch 动作解码、Zarr replay buffer，支持单任务/多任务训练与 `dexmani_sim` 仿真评测。

> 阅读顺序：01 → 02 → 03 → 04 → 05 → 06 → 07
> 快速导航：第 1 节 核心能力 · 第 3 节 最小命令 · 第 5 节 文档索引 · 第 8 节 FAQ

---

## 1. 核心能力

| 能力 | 说明 |
|---|---|
| 感知模态 | RGB（单视角）、Point Cloud（1024 点 × 3/6 维）、Joint State（19 维） |
| 动作解码 | Diffusion（DDPM/DDIM）、FlowMatch（Rectified Flow + Consistency） |
| 策略架构 | DP、DP3、ManiFlow、MoE DP3、MultiTask DiT |
| 训练模式 | 单卡 (train.py)、多卡 DDP (train_ddp.py)、多任务 (train_multi_task.py) |
| 仿真评测 | 7 个灵巧手操作任务，支持训练期内评估 + 独立评测脚本 |
| 实验管理 | Wandb 日志、TopK checkpoint、Hydra 输出目录自动组织 |

### 不变约束

```
horizon = 16        n_obs_steps = 2       n_action_steps = 8       action_dim = 19
```

所有策略共享此约束：策略接收 2 步历史观察，预测 16 步动作轨迹，实际执行前 8 步，动作空间对应 XArm7（7 DoF）+ XHand（12 DoF）= 19 维绝对关节角。

## 2. 环境搭建

### 2.1 依赖

- Python 3.10+
- CUDA 11.8+ / 12.x
- PyTorch 2.x
- Conda env: `policy`

### 2.2 安装

```bash
# 激活环境
conda activate policy

# 安装 dexmani_policy（开发模式）
pip install -e .

# 安装 dexmani_sim（仿真评测依赖，外部包）
cd ~/Desktop/dexmani_sim && pip install -e .
```

### 2.3 数据准备

训练数据为 Zarr 格式，存放在 `robot_data/sim/` 下：

```
robot_data/
└── sim/
    ├── pick_apple_messy.zarr
    ├── pour.zarr
    ├── multi_grasp.zarr
    ├── place_milk_box.zarr
    ├── open_box.zarr
    ├── pick_bottle.zarr
    └── stack_cups.zarr
```

每个 `.zarr` 文件包含 `joint_state`、`action` 及对应传感器数据（rgb / point_cloud）。

## 3. 最小可运行命令

### 3.1 单卡训练

```bash
# 使用 dp3 策略训练 pick_apple_messy 任务
bash scripts/train.sh dp3

# Hydra override：修改训练参数
bash scripts/train.sh dp3 'training.loop.num_epochs=100' 'optimizer.lr=3e-5'
```

支持的策略名：`dp` | `dp3` | `maniflow` | `moe_dp3`（对应 `configs/` 下的同名 yaml）

### 3.2 多卡 DDP 训练

```bash
# 使用 maniflow_ddp 配置（4 卡）
bash scripts/train_ddp.sh maniflow_ddp

# 指定 GPU 和 override
bash scripts/train_ddp.sh maniflow_ddp 'training.num_gpus=2' 'training.gpu_ids=[0,1]'
```

### 3.3 多任务训练

```bash
bash scripts/train_multi_task.sh multitask_dit
```

### 3.4 仿真评测

```bash
# 对指定实验目录评测
bash scripts/eval_sim.sh dp3 pick_apple_messy 2026-04-01_11-18_233

# 指定 checkpoint 和 denoise 步数
bash scripts/eval_sim.sh dp3 pick_apple_messy 2026-04-01_11-18_233 \
    'eval.offline.ckpt_tag_or_path=best' \
    'eval.offline.denoise_timesteps_list=[5,10,20]'
```

### 3.5 Wandb 日志同步

```bash
bash scripts/wandb_sync.sh
```

### 3.6 冒烟测试

```bash
# 单 config 测试（验证 config→dataset→model→forward→backward 链路）
python dexmani_policy/smoke_test.py dp3

# 批量测试
python dexmani_policy/smoke_test.py dp3 maniflow moe_dp3
```

## 7. 关键约定速查

| 约定 | 说明 |
|---|---|
| Normalizer | `mode='limits'`，在所有数据（train+val）上拟合，缩放到 [-1,1] |
| 数据增强 | 默认禁用，配置中注释掉的 block 取消注释即可启用 |
| Wandb 模式 | 默认 `offline`，需手动 `wandb sync` |
| Checkpoint 格式 | `simple.v1`，包含 `{state, weights}` |
| `latest.pt` | 指向最新 checkpoint 的符号链接 |
| TopK 跟踪 | 默认 `topk=3`，monitor `test_mean_score`，mode `max` |
| DDP checkpoint 兼容 | `fix_state_dict()` 自动处理 `module.` 前缀转换 |
| CLIP text encoder | 冻结，仅 `text_proj` 可训练 |
| MoE aux loss | 全程生效，无 warmup 阶段 |
| 评测依赖 | `dexmani_sim` 动态导入，`importlib.import_module(f"dexmani_sim.envs.{task_name}")` |

## 8. 常见问题

**Q: 如何新增一个任务？**
1. 准备 Zarr 数据 → `robot_data/sim/<task_name>.zarr`
2. 修改配置中 `task_name` 和 `zarr_path`
3. 如果在 `dexmani_sim` 中有对应环境，设置 `env_runner.task_name`

**Q: 如何启用数据增强？**
取消配置文件中 `augmentation_cfg` block 的注释，注意 RGB 增强需 `sensor_modalities` 包含 `rgb`，PC 颜色增强需 `pc_dim >= 6`

**Q: 单卡 checkpoint 能用于 DDP 恢复吗？**
能。`fix_state_dict()` 自动处理 `module.` 前缀的添加/移除

**Q: 训练中断后如何继续？**
直接重新运行相同命令，Trainer 会自动 `load_for_resume("latest")`

**Q: 评测时使用哪个 checkpoint？**
- `eval.offline.ckpt_tag_or_path: best` → `topk_manifest.json` 中 score 最高的
- `eval.offline.ckpt_tag_or_path: latest` → `latest.pt` 符号链接
- 也可以传具体文件名如 `epoch=0100-step=0012345-score=0.7500.pt`
