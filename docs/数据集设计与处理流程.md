# 数据集设计与处理流程

## 概述

dexmani_policy 的数据集系统基于 **Zarr 格式的 episode 回放缓冲区**，支持多模态观测（点云、RGB、深度、本体感知）和灵活的时序采样策略。设计核心思想是：

1. **Episode 级存储**：按轨迹（episode）组织数据，支持变长轨迹
2. **时序窗口采样**：训练时采样固定长度的时序片段（horizon）
3. **模态解耦**：不同传感器模态独立存储和处理
4. **训练/验证隔离**：episode 级划分，确保评估无数据泄露
5. **增强可选**：支持模态特定的数据增强（RGB、点云）

## 一、数据存储格式

### 1.1 Zarr 目录结构

```
dataset.zarr/
├── meta/
│   └── episode_ends          # [n_episodes] int64，累积时间步索引
└── data/
    ├── joint_state           # [total_steps, 19] float32
    ├── action                # [total_steps, 19] float32
    ├── point_cloud           # [total_steps, N, 6] float32 (xyz + rgb)
    ├── rgb                   # [total_steps, H, W, 3] uint8
    ├── depth                 # [total_steps, H, W] uint16 (毫米)
    ├── camera_intrinsic      # [total_steps, 3, 3] float32
    └── camera_extrinsic      # [total_steps, 3, 4] 或 [4, 4] float32
```

**关键设计**：
- `episode_ends` 记录每个 episode 的结束索引，例如 `[100, 250, 400]` 表示 3 个 episode，长度分别为 100、150、150
- 所有数据按时间步（timestep）展平存储，通过 `episode_ends` 索引切分
- 支持分块压缩（默认 2MB chunk），高效读取和存储

### 1.2 数据类型约定

| 模态 | 形状 | 数据类型 | 值域 | 说明 |
|------|------|----------|------|------|
| `joint_state` | `[T, 19]` | `float32` | 关节角度/速度 | 本体感知状态 |
| `action` | `[T, 19]` | `float32` | 关节目标 | 动作指令 |
| `point_cloud` | `[T, N, 6]` | `float32` | xyz: 米, rgb: [0,1] | 点云 + 颜色 |
| `rgb` | `[T, H, W, 3]` | `uint8` | [0, 255] | RGB 图像 |
| `depth` | `[T, H, W]` | `uint16` | [0, 65535] 毫米 | 深度图 |
| `camera_intrinsic` | `[T, 3, 3]` | `float32` | 相机内参矩阵 | fx, fy, cx, cy |
| `camera_extrinsic` | `[T, 3, 4]` 或 `[T, 4, 4]` | `float32` | 相机外参 | 旋转 + 平移 |

## 二、数据集类层次结构

```
BaseDataset (基类)
├── PCDataset (点云 + 本体感知)
├── RGBDataset (RGB + 本体感知)
├── RGBPCDataset (RGB + 深度 + 点云 + 本体感知)
└── MultiTaskDataset (多任务混合采样)
```

### 2.1 BaseDataset

**核心职责**：
- 加载 Zarr 数据并验证必需字段
- 创建 `SequenceSampler` 进行时序窗口采样
- 划分训练集/验证集（episode 级）
- 管理 `LinearNormalizer` 进行数据归一化

**关键参数**：
```python
BaseDataset(
    zarr_path='data/task.zarr',
    seed=42,
    horizon=16,              # 时序窗口长度
    pad_before=0,            # 窗口前填充
    pad_after=0,             # 窗口后填充
    val_ratio=0.1,           # 验证集比例
    max_train_episodes=None, # 训练集最大 episode 数
    sensor_modalities=['joint_state', 'point_cloud'],
    augmentation_cfg=None,
)
```

**数据验证**：
```python
# 初始化时检查必需字段
required_keys = set(sensor_modalities + ['action'])
available_keys = set(self.replay_buffer.keys())
missing_keys = required_keys - available_keys

if missing_keys:
    raise ValueError(
        f"Zarr file missing required keys: {sorted(missing_keys)}\n"
        f"Required: {sorted(required_keys)}\n"
        f"Available: {sorted(available_keys)}\n"
        f"Path: {zarr_path}"
    )
```

### 2.2 子类扩展

**PCDataset**：
- 输出点云 `(T, N, 6)` float32，xyz + rgb
- 可选点云颜色增强（`PCAug`）

**RGBDataset**：
- 输出 RGB `(T, H, W, 3)` uint8
- 可选 RGB 增强（`RGBAug`，ColorJitter）

**RGBPCDataset**：
- 输出 RGB、深度、点云、相机参数
- 支持 RGB 和点云双增强

**MultiTaskDataset**：
- 混合多个任务的数据集
- 支持 3 种采样策略：`balanced`（均衡）、`weighted`（加权）、`proportional`（按数据量比例）
- 验证集使用确定性顺序遍历（`_deterministic=True`）

## 三、数据处理流程

### 3.1 训练数据流

```
Zarr 文件
  ↓
ReplayBuffer.copy_from_path()  # 加载数据
  ↓
SequenceSampler                # 生成时序窗口索引
  ↓
BaseDataset.__getitem__()      # 采样单个 batch
  ↓
子类处理（增强、格式转换）
  ↓
LinearNormalizer.normalize()   # 归一化
  ↓
返回 {'obs': {...}, 'action': [...]}
```

### 3.2 SequenceSampler 采样逻辑

**目标**：从 episode 中安全地切出固定长度的时序片段。

**核心算法**（`create_indices`）：
```python
for each episode in episode_mask:
    episode_length = episode_ends[i] - episode_ends[i-1]
    
    # 计算有效起始位置范围
    min_start = -pad_before
    max_start = episode_length - sequence_length + pad_after
    
    # 检查 episode 是否足够长
    if max_start < min_start:
        raise ValueError("Episode too short")
    
    # 生成所有有效窗口
    for idx in range(min_start, max_start+1):
        buffer_start_idx = max(idx, 0) + start_idx
        buffer_end_idx = min(idx+sequence_length, episode_length) + start_idx
        sample_start_idx = 0 + start_offset
        sample_end_idx = sequence_length - end_offset
        indices.append([buffer_start_idx, buffer_end_idx, sample_start_idx, sample_end_idx])
```

**边界处理**：
- `pad_before/pad_after` 允许窗口超出 episode 边界
- 超出部分通过重复边界帧填充（`np.pad(..., mode='edge')`）

**示例**：
```
episode_length = 10, horizon = 5, pad_before = 2, pad_after = 1

有效起始位置: [-2, -1, 0, 1, 2, 3, 4, 5, 6]
  idx=-2: [0,0,0,1,2] (前2帧重复)
  idx=-1: [0,0,1,2,3] (前1帧重复)
  idx=0:  [0,1,2,3,4]
  ...
  idx=6:  [6,7,8,9,9] (后1帧重复)
```

### 3.3 训练/验证划分

**Episode 级划分**：
```python
# 随机选择 val_ratio 比例的 episode 作为验证集
val_mask = get_val_mask(seed=seed, val_ratio=0.1, n_episodes=100)
train_mask = ~val_mask

# 可选：限制训练集 episode 数量
train_mask = downsample_mask(seed=seed, mask=train_mask, max_n=50)
```

**验证集创建**：
```python
def get_validation_dataset(self):
    val_set = copy.deepcopy(self)
    val_set.sampler = SequenceSampler(
        replay_buffer=self.replay_buffer,  # 共享 buffer 节省内存
        sequence_length=self.horizon,
        pad_before=self.pad_before,
        pad_after=self.pad_after,
        episode_mask=self.val_mask,        # 使用验证集 mask
    )
    val_set.augmentation_cfg = None        # 验证集不增强
    return val_set
```

**关键特性**：
- 训练集和验证集完全隔离，无数据泄露
- 验证集不使用数据增强
- 共享 `replay_buffer` 节省内存

## 四、数据归一化

### 4.1 LinearNormalizer

**设计思想**：将数据线性映射到 `[-1, 1]` 区间。

**支持模式**：
- `limits`：根据关节限位归一化（用于 `joint_state` 和 `action`）
- `gaussian`：标准化到均值 0、标准差 1
- `identity`：不归一化（用于点云、RGB、相机参数）

**示例**：
```python
normalizer = LinearNormalizer()

# joint_state: 使用关节限位
normalizer['joint_state'] = LinearNormalizer.create_fit(
    data=joint_state_data,
    mode='limits',
    limits_low=robot.joint_limits_low,
    limits_high=robot.joint_limits_high,
)

# action: 同样使用限位
normalizer['action'] = normalizer['joint_state']

# point_cloud: 不归一化
normalizer['point_cloud'] = LinearNormalizer.create_identity()
```

### 4.2 归一化时机

**训练时**：
```python
# Dataset.__getitem__() 返回原始数据
sample = {'obs': {'joint_state': [...], 'point_cloud': [...]}, 'action': [...]}

# Trainer 中归一化
nobs = normalizer.normalize(sample['obs'])
nactions = normalizer['action'].normalize(sample['action'])
```

**推理时**：
```python
# 观测归一化
nobs = normalizer.normalize(obs_dict)

# 预测动作
naction = policy.predict_action(nobs)

# 反归一化
action = normalizer['action'].unnormalize(naction)
```

## 五、数据增强

### 5.1 RGB 增强（RGBAug）

**位置**：`dexmani_policy/datasets/augmentation/rgb_aug.py`

**策略**：
- `ColorJitter`：亮度、对比度、饱和度、色调随机扰动
- **时序一致性**：同一 episode 内使用相同的增强参数

**实现**：
```python
class RGBAug:
    def __init__(self, brightness=0.2, contrast=0.2, saturation=0.2, hue=0.1):
        self.transform = transforms.ColorJitter(brightness, contrast, saturation, hue)
    
    def __call__(self, images):
        # images: (T, H, W, 3) uint8
        T = images.shape[0]
        params = self.transform.get_params(...)  # 采样一次增强参数
        
        augmented = []
        for t in range(T):
            img = Image.fromarray(images[t])
            img = transforms.functional.apply_jitter(img, *params)  # 使用相同参数
            augmented.append(np.array(img))
        
        return np.stack(augmented, axis=0)
```

### 5.2 点云增强（PCAug）

**位置**：`dexmani_policy/datasets/augmentation/pc_aug.py`

**策略**：
- **颜色扰动**：RGB 通道添加高斯噪声
- **时序一致性**：同一 episode 内使用相同的噪声

**实现**：
```python
class PCAug:
    def __init__(self, color_std=0.05):
        self.color_std = color_std
    
    def __call__(self, point_cloud):
        # point_cloud: (T, N, 6) float32, xyz + rgb
        T, N, _ = point_cloud.shape
        
        # 采样一次噪声，广播到所有时间步
        noise = np.random.randn(1, N, 3) * self.color_std
        point_cloud[..., 3:6] = np.clip(point_cloud[..., 3:6] + noise, 0, 1)
        
        return point_cloud
```

### 5.3 增强配置

**配置示例**（`dp3.yaml`）：
```yaml
dataset:
  _target_: dexmani_policy.datasets.pc_dataset.PCDataset
  augmentation_cfg:
    pc:
      color_std: 0.05  # 点云颜色扰动标准差
```

**接口统一**：
- 所有子类通过 `augmentation_cfg` 字典控制增强
- 验证集自动禁用增强（`augmentation_cfg=None`）

## 六、多任务数据集

### 6.1 MultiTaskDataset 设计

**目标**：混合多个任务的数据，支持灵活的采样策略。

**采样策略**：
1. **balanced**：每个任务采样概率相等
   ```python
   sample_probs = [1/N, 1/N, ..., 1/N]
   ```

2. **weighted**：根据 `task_weights` 指定概率
   ```python
   sample_probs = task_weights / sum(task_weights)
   ```

3. **proportional**：按数据量比例采样
   ```python
   sample_probs = task_lengths / sum(task_lengths)
   ```

### 6.2 确定性验证集

**问题**：训练集使用动态采样，验证集也动态采样会导致评估结果不稳定。

**解决方案**：验证集使用固定索引映射。

```python
class MultiTaskDataset:
    def __init__(self, ..., _deterministic=False):
        self._deterministic = _deterministic
        
        if self._deterministic:
            # 预生成固定的 (task_idx, local_idx) 映射
            self._fixed_indices = []
            for task_idx, task_length in enumerate(self.task_lengths):
                for local_idx in range(task_length):
                    self._fixed_indices.append((task_idx, local_idx))
    
    def __getitem__(self, idx):
        if self._deterministic:
            # 确定性模式：按固定顺序返回
            task_idx, local_idx = self._fixed_indices[idx]
        else:
            # 动态采样模式（训练集）
            item_seed = hash((self.seed, self._epoch, idx)) % (2**32)
            item_rng = np.random.default_rng(item_seed)
            task_idx = item_rng.choice(self.num_tasks, p=self.sample_probs)
            local_idx = item_rng.integers(0, self.task_lengths[task_idx])
        
        sample = self.datasets[task_idx][local_idx]
        sample['task_id'] = task_idx
        sample['task_name'] = self.task_names[task_idx]
        return sample
    
    def get_validation_dataset(self):
        val_datasets = [d.get_validation_dataset() for d in self.datasets]
        return MultiTaskDataset(
            datasets=val_datasets,
            task_names=self.task_names,
            sampling_strategy='proportional',
            normalizer_mode=self.normalizer_mode,
            seed=self.seed,
            _deterministic=True,  # ← 验证集启用确定性模式
        )
```

## 八、最佳实践

### 8.1 数据收集

1. **Episode 长度**：建议 50-200 步，过短难以学习长期依赖，过长增加存储和采样开销
2. **数据量**：单任务至少 50-100 个 episode，多任务每个任务至少 30 个
3. **验证集比例**：10-20%，确保有足够的验证样本
4. **深度数据类型**：使用 `uint16` 存储毫米级深度，范围 0-65535

### 8.2 超参数调优

1. **horizon**：通常 8-16，取决于任务复杂度和动作频率
2. **pad_before/pad_after**：允许窗口超出 episode 边界，增加数据多样性
3. **augmentation_cfg**：RGB 增强强度 0.1-0.3，点云颜色扰动 0.03-0.1
4. **max_train_episodes**：限制训练集大小，加速迭代和调试

### 8.3 调试技巧

1. **数据验证**：初始化时自动检查必需字段，提供详细错误信息
2. **可视化**：使用 `replay_buffer.get_episode(i)` 查看单个 episode
3. **采样测试**：检查 `sampler.indices` 确认窗口边界正确
4. **归一化检查**：确认归一化后数据在 `[-1, 1]` 范围内

## 九、常见问题

**Q1: 为什么使用 episode 级划分而不是时间步级？**

A: Episode 级划分确保训练集和验证集的轨迹完全独立，避免时序相关性导致的数据泄露。时间步级划分会导致相邻时间步分布在训练集和验证集，模型可能记住验证集的上下文。

**Q2: 为什么验证集不使用数据增强？**

A: 验证集的目的是评估模型在真实数据上的泛化能力，增强会引入额外的随机性，导致评估结果不稳定。训练集使用增强是为了提高鲁棒性。

**Q3: MultiTaskDataset 的 `_deterministic` 模式有什么用？**

A: 训练集使用动态采样（每个 epoch 采样不同的样本），验证集使用确定性顺序遍历（每次评估看到相同的样本），确保评估结果可复现和可比较。

**Q4: 为什么点云和 RGB 不归一化？**

A: 点云的 xyz 坐标和 RGB 的颜色值已经在合理的范围内（米级坐标、[0,1] 颜色），归一化反而会破坏物理意义。关节状态和动作需要归一化是因为不同关节的范围差异很大。

**Q5: 如何处理变长的点云？**

A: 当前实现假设点云数量固定（通过下采样或填充）。如果需要支持变长点云，可以在 Dataset 中添加 `valid_mask` 字段，或使用 padding + attention mask。

dexmani_policy 的数据集系统通过以下设计实现了高效、灵活、可靠的数据处理：

2. **Episode 组织**：支持变长轨迹和 episode 级划分

4. **模态解耦**：独立的处理和增强管道
5. **归一化管理**：统一的归一化接口和模式
6. **多任务支持**：灵活的采样策略和确定性验证

这套系统为灵巧操作模仿学习提供了坚实的数据基础。
