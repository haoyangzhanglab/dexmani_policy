```
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Zarr 磁盘文件 (robot_data/sim/<task>.zarr)                                  │
│    │                                                                        │
│    ▼                                                                        │
│  ReplayBuffer.copy_from_path()    ← 全量加载为 numpy 数组 (内存)              │
│    │                                                                        │
│    ├── data/                                                                │
│    │   ├── joint_state  (N, 19)                                             │
│    │   ├── action        (N, 19)                                            │
│    │   ├── point_cloud   (N, 1024, 3|6)  ← 可选                             │
│    │   ├── rgb           (N, 224, 224, 3) ← 可选                            │
│    │   └── ...                                                              │
│    └── meta/                                                                │
│        └── episode_ends  (E,)            ← 每个 episode 的结束帧索引          │
│    │                                                                        │
│    ▼                                                                        │
│  get_val_mask(n_episodes=n_episodes, val_ratio=val_ratio, seed=seed)        │
│    ├── val_mask (numpy bool array)                                           │
│    └── train_mask = ~val_mask → downsample_mask(max_n=max_train_episodes)    │
│    │                                                                        │
│    ▼                                                                        │
│  SequenceSampler(replay_buffer, horizon=16, pad_before=1, pad_after=7,      │
│                  episode_mask=train_mask)                                   │
│    └── create_indices() ← numba JIT 编译                                    │
│        ├── pad_before/after 被 clamp 到 [0, sequence_length-1]              │
│        └── 滑动窗口: 每个 episode 内起始偏移从 -pad_before 到                  │
│            (len - horizon + pad_after)，边界用首/末帧填充                      │
│    │                                                                        │
│    ▼                                                                        │
│  BaseDataset.__getitem__(idx)                                               │
│    ├── sampler.sample_sequence(idx) → numpy sample (obs + action)           │
│    ├── sample_to_data(sample) → {'obs': {...}, 'action': ...}               │
│    ├── apply_augmentation(data) → 按概率对各模态施加增强                       │
│    └── dict_apply(data, torch.from_numpy) → Tensor                          │
│    │                                                                        │
│    ▼                                                                        │
│  DataLoader (shuffle=True, batch_size=256, num_workers=8)                    │
│    └── collate: 默认在 dim=0 堆叠 → batch (B, 16, *)                        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

> **关于 `persistent_workers` 的内存占用**：所有策略配置默认 train dataloader `persistent_workers=true, num_workers=8`（val dataloader 使用 `persistent_workers=false, num_workers=4`）。DataLoader 的每个 worker 进程持有完整的 `replay_buffer` 副本（numpy 数组），因此实际内存占用 ≈ `num_workers × dataset_size`。以典型数据集（~500MB）为例，8 个 worker 额外占用约 4GB 内存。对于大数据集或多任务训练，建议适当减少 `num_workers` 或将 `persistent_workers` 设为 `false`（以每次 epoch 重建 worker 为代价换取更低内存）。

## 数据存储格式 — Zarr + ReplayBuffer

### 各策略对模态数据的 Shape 与 Dtype 要求

以下从 Zarr 存储 → Dataset 采样 → DataLoader 批处理 → Agent 预处理四个阶段，逐策略列出所需的模态 key 及其 shape/dtype 约束。

#### 策略-模态需求矩阵

| 策略 | 必需模态 | 可选模态 |
|---|---|---|
| DPAgent | `rgb`, `joint_state` | — |
| DP3Agent | `point_cloud`, `joint_state` | — |
| MoEAgent | `point_cloud`, `joint_state` | — |
| ManiFlowAgent | `point_cloud`, `joint_state` | — |
| MultiTaskAgent | `rgb`, `joint_state`, `task_text`(str) | — |

#### 各模态在各阶段的 Shape 与 Dtype

**point_cloud** (DP3Agent / MoEAgent / ManiFlowAgent):

```
阶段          Shape                    Dtype      说明
─────────────────────────────────────────────────────────────
Zarr 存储     (N, 1024, C)            float32    C=3 (仅XYZ) 或 C=6 (XYZ+RGB归一化到[0,1])
Sample        (16, 1024, C)           float32    16 帧窗口
Batch         (B, 16, 1024, C)        float32    DataLoader 默认 stack
Preprocessed  (B×2, 1024, C)          float32    截断 n_obs_steps=2 + flatten → FPS 下采样到 num_points
```

- `pc_dim` 配置必须与 Zarr 中的 C 匹配：`pc_dim=3` 时仅使用 XYZ，`pc_dim=6` 时使用 XYZ+RGB
- `use_coord_only` 标志 (DP3/MoE/ManiFlow 的 ObsEncoder) 在 `pc_dim=3` 时仅取 `[..., :3]`
- FPS 下采样: 若 `pc.shape[1] > num_points`，随机采样到 `num_points` 个点
- ManiFlowAgent 额外要求: PC 经 patch tokenizer 编码为 K+1 个 token (1 global + K patches)

**rgb** (DPAgent / MultiTaskAgent):

```
阶段          Shape                    Dtype      说明
─────────────────────────────────────────────────────────────
Zarr 存储     (N, H, W, 3)            uint8      原始 H×W 可变
Sample        (16, H, W, 3)           uint8      16 帧窗口
Batch         (B, 16, H, W, 3)        uint8      DataLoader 默认 stack
Preprocessed  (B×2, 3, 224, 224)      float32    ImageProcessor: uint8→float32÷255, resize, center crop, normalize
```

- Zarr 中 RGB 必须为 `uint8` 类型，`to_rgb_tensor()` 自动除以 255 并转为 float32
- 若 Zarr 中已是 float32 且值域在 [0,1]，也会被接受
- 通道顺序: ImageProcessor 自动处理 HWC↔CHW 转换，Zarr 中同时支持 (H,W,3) 和 (3,H,W)
- ImageProcessor 根据 backbone preset 使用不同的 mean/std 和 interpolation (见 `IMAGE_PROCESSOR_PRESETS`)

**joint_state** (所有策略):

```
阶段          Shape                    Dtype      说明
─────────────────────────────────────────────────────────────
Zarr 存储     (N, 19)                 float32    19 维关节角度
Sample        (16, 19)                float32    16 帧窗口
Batch         (B, 16, 19)             float32    DataLoader 默认 stack
Preprocessed  (B×2, 19)               float32    截断 n_obs_steps=2 + flatten + normalize 到 [-1,1]
```

- `state_dim=19` 是核心不变约束，必须与 Zarr 中 joint_state 的最后一维一致
- Normalizer 逐维度独立缩放 (last_n_dims=1)，低方差维度 (range < 2e-2) 平移到零均值，不做缩放，遵循 Diffusion Policy 附录建议，避免数值放大

**action** (所有策略):

```
阶段          Shape                    Dtype      说明
─────────────────────────────────────────────────────────────
Zarr 存储     (N, 19)                 float32    ReplayBuffer 强制转换 (若非 float32)
Sample        (16, 19)                float32    16 帧窗口
Batch         (B, 16, 19)             float32    DataLoader 默认 stack
Preprocessed  (B, 16, 19)             float32    normalizer['action'].normalize() → [-1,1]
```

- `action_dim=19` 是核心不变约束
- `ReplayBuffer.copy_from_store()` 中所有浮点类型数组统一转换为 `float32`（不仅是 action），避免 point_cloud/rgb 以 float64 进入 encoder 造成显存翻倍

**depth** (RGBPCDataset):

```
阶段          Shape                    Dtype      说明
─────────────────────────────────────────────────────────────
Zarr 存储     (N, H, W) 或 (N, H, W, 1)  uint16/float32
Sample        (16, H, W) 或 (16, H, W, 1)  → float32  to_depth_tensor() 统一转为 (N, 1, H, W)
```

**camera_intrinsic / camera_extrinsic** (RGBPCDataset):

```
阶段          Shape                    Dtype      说明
─────────────────────────────────────────────────────────────
Zarr 存储     (N, 3, 3) / (N, 4, 4)   float32    不参与归一化 (identity normalizer)
```

**task_text / task_name** (MultiTaskAgent):

```
阶段          类型          说明
─────────────────────────────────────────────
Zarr 存储     无 (元数据)   来自 MultiTaskDataset 配置
Sample        str           由 MultiTaskDataset.__getitem__ 注入
Batch         List[str]     DataLoader 默认 collate 聚合为 list，长度 B
Preprocessed  (B, n_emb)    CLIP tokenize → text_proj → 拼接到 cond 尾部
```

- `MultiTaskAgent` 要求 `obs_dict` 中必须包含 `task_text` key，否则抛出 `ValueError`
- `task_name` 仅供 env_runner 使用，不参与前向计算

#### 模态缺失容忍度

| 策略 | 缺失某模态时的行为 |
|---|---|
| DPAgent | 缺少 `rgb` 或 `joint_state` → KeyError (直接崩溃) |
| DP3Agent / MoEAgent | 缺少 `point_cloud` 或 `joint_state` → KeyError |
| ManiFlowAgent | 缺少 `point_cloud` 或 `joint_state` → KeyError |
| MultiTaskAgent | 缺少 `task_text` → ValueError (显式检查)；缺少 `rgb` → KeyError |
| 所有策略 (modality_dropout) | 按 `modality_dropout_probs` 概率将该模态置零（仅对 normalizer 中存在的 key 生效，非归一化模态不受影响），不会因缺失崩溃 |

```
<task>.zarr/

│   ├── joint_state   (N, 19)          float32   ← 关节角度

│   └── camera_extrinsic  (N, 4, 4)    float32   ← 可选

```

### ReplayBuffer

`ReplayBuffer` (`datasets/common/replay_buffer.py:82`) 是 Zarr 数据的运行时容器。

| 操作 | 方法 | 说明 |
|---|---|---|
| 全量加载到内存 | `copy_from_path(path)` | 默认方式，将 Zarr 数据全量读为 numpy 数组，后续随机访问延迟最低 |
| 磁盘直接打开 | `create_from_path(path, mode='r')` | 大数据集用，每次采样从磁盘读取 |
| 动态写入 | `add_episode(data)` | 增量追加 episode，采集数据时使用 |
| 随机访问 | `get_episode(idx)` / `get_steps_slice(start, stop)` | 按 episode 或时间步读取 |

**核心属性**:
- `n_steps` — 总帧数 (`episode_ends[-1]`)
- `n_episodes` — 总 episode 数 (`len(episode_ends)`)
- `episode_ends` — 累积边界数组，episode i 覆盖帧索引 `[ends[i-1], ends[i])`
- `episode_lengths` — 各 episode 的长度

**压缩配置**: 默认使用 `numcodecs.Blosc(cname='lz4', clevel=5, shuffle=numcodecs.Blosc.NOSHUFFLE)`，action 若非 `float32` 则自动转换为 `float32`。

## 序列采样机制 — SequenceSampler

`SequenceSampler` (`datasets/common/sampler.py:87`) 将不定长 episode 切分为固定长度训练序列。

### 核心参数

| 参数 | 默认值 | 说明 |
|---|---|---|
| `sequence_length` (horizon) | 16 | 序列长度 |
| `pad_before` | `n_obs_steps - 1 = 1` | 序列起始前的 padding 帧数，`create_indices()` 内 clamp 到 `[0, sequence_length-1]` |
| `pad_after` | `n_action_steps - 1 = 7` | 序列结束后的 padding 帧数，`create_indices()` 内 clamp 到 `[0, sequence_length-1]` |
| `episode_mask` | train_mask / val_mask | 控制哪些 episode 参与采样 |

### 滑动窗口算法

`create_indices()` 是 numba JIT 编译的函数：

```
对每个 episode (长度 L):
  min_start = -pad_before           (= -1)
  max_start = L - horizon + pad_after (= L - 16 + 7 = L - 9)

  for idx in [min_start, max_start]:
    buffer 区间: [max(idx,0), min(idx+16, L))
    边界填充:    起始不足 → 用首帧填充; 末端不足 → 用末帧填充
```

**示意图** (horizon=16, pad_before=1, pad_after=7):

```
episode 内部帧:  [0] [1] [2] ... [L-3] [L-2] [L-1]

序列起始 idx=-1: [0][0][1][2]...[14]     ← 填充 1 帧（pad_before=1）
序列起始 idx=0:  [0][1][2]...[15]        ← 无填充
序列起始 idx=L-16: [L-16]...[L-1]        ← 无填充
序列起始 idx=L-9:  [L-9]...[L-1][L-1]x7  ← 填充 7 帧（pad_after=7），max_start
```

**说明**：
- `idx=-1` 时，buffer 读取 `[0, 15)`，sample 填充 `[0:1]` 用首帧 → `[0][0][1]...[14]`
- `idx=L-9` 时，buffer 读取 `[L-9, L)`，sample 填充 `[9:16]` 用末帧 → `[L-9]...[L-1][L-1]x7`

**最小 episode 需求**: `L >= sequence_length - pad_before - pad_after = 16 - 1 - 7 = 8` 帧。

### 采样输出

`sample_sequence(idx)` 返回 `{key: numpy (16, *)}`，数据填充逻辑：

1. 从 replay_buffer 读取 `[buffer_start, buffer_end)` 的实际帧
2. 若 `sample_start > 0`（左侧越界），用首帧填充前 `sample_start` 帧
3. 若 `sample_end < 16`（右侧越界），用末帧填充后 `16-sample_end` 帧

## 数据集变体

### BaseDataset

`datasets/base_dataset.py:10` — 所有单任务数据集的基类。

```
BaseDataset(zarr_path, seed, horizon, pad_before, pad_after, val_ratio,
            max_train_episodes, sensor_modalities, augmentation_cfg)
```

**核心流程**:

```
__init__:
  1. ReplayBuffer.copy_from_path(zarr_path, keys=sensor_modalities + ['action'])
                                              → 加载数据（仅加载声明的模态 + action）
  2. 设置 self.augmentation_cfg，若非 None 则调用 _build_augmentors()
                                              → 构建增强器 (子类实现)
  3. val_mask = get_val_mask(n_episodes, val_ratio, seed)
  4. train_mask = ~val_mask → downsample_mask(max_n=max_train_episodes)
                                              → 切分并下采样训练集 episode
  5. SequenceSampler(replay_buffer, horizon, pad_before, pad_after,
                     episode_mask=train_mask)  → 构建训练集采样器

__getitem__:
  1. sampler.sample_sequence(idx)              → numpy 数组
  2. sample_to_data(sample)                     → 组织成 {obs, action}
  3. apply_augmentation(data)                   → 按概率增强
  4. dict_apply(torch.from_numpy)               → 转换为 Tensor

get_normalizer():
  对 joint_state + action 拟合 LinearNormalizer (mode='limits')
```

**训练/验证集切分**:
- `val_ratio=0.05`: 按 episode 级别切分，默认 5% 的 episode 作为验证集
- `max_train_episodes`: 限制训练集 episode 数（通过 `downsample_mask` 随机采样）
- `get_validation_dataset()`: 返回 `copy.copy(self)` 浅拷贝，共享 replay_buffer，但 sampler 使用 val_mask，同时设置 `augmentation_cfg=None` 和 `augmentors={}` 以断开增强引用。若 `val_mask` 全为 False（如 `val_ratio=0`），返回 `None`

| 特性 | PCDataset | RGBDataset | RGBPCDataset |

| 感知模态 | point_cloud + joint_state | rgb + joint_state | rgb + depth + point_cloud + joint_state + camera |
| PC 增强 | color / spatial / dropout | — | color / spatial / dropout |

| Normalizer 拟合 | joint_state + action | joint_state + action | joint_state + action + camera(I) |
| 适用 Agent | DP3Agent, MoEAgent, ManiFlowAgent | DPAgent, MultiTaskAgent | — |

### Agent-Dataset 选配指南

#### 兼容性矩阵

| Agent | PCDataset | RGBDataset | RGBPCDataset | MultiTaskDataset |
|---|---|---|---|---|
| DPAgent | **错配** — 缺少 `rgb` → KeyError | **正确** | — | — |
| DP3Agent | **正确** | **错配** — 缺少 `point_cloud` → KeyError | — | — |
| MoEAgent | **正确** | **错配** — 缺少 `point_cloud` → KeyError | — | — |
| ManiFlowAgent | **正确** | **错配** — 缺少 `point_cloud` → KeyError | — | — |
| MultiTaskAgent | — | **正确** (作为子数据集) | — | **正确** |

#### 配错时的典型错误信息

| 错配场景 | 典型报错 |
|---|---|
| DPAgent + PCDataset | `KeyError: 'rgb'` — DPObsEncoder 尝试访问 `obs['rgb']` |
| DP3Agent + RGBDataset | `KeyError: 'point_cloud'` — DP3ObsEncoder 尝试访问 `obs['point_cloud']` |

## 数据归一化 — LinearNormalizer

`common/normalizer.py:242` — 所有数据集共用同一套归一化机制（`LinearNormalizer`）。

### 拟合模式

| 模式 | 公式 | 适用场景 |
|---|---|---|
| `limits` | `x_norm = (x - min) / (max - min) * 2 - 1` | 有明确物理边界的量 (关节角度、动作) |
| `gaussian` | `x_norm = (x - mean) / std` | 分布近似正态的量 |

当前全部使用 `mode='limits'`，缩放到 `[-1, 1]`。

### 拟合流程

```
dataset.get_normalizer(mode='limits'):
  1. 从 replay_buffer 提取 joint_state + action 全量数据 (train+val)
  2. fit_params(data, last_n_dims=1):
       - 将数据 reshape 为 (-1, dim)，逐维度计算 min/max
       - scale = 2.0 / (max - min)
       - offset = -1.0 - scale * min
       - 对于 range < range_eps 的低方差维度，平移到零均值（scale=1, offset=-mean），不做缩放，避免数值放大
  3. 返回 LinearNormalizer(params_dict)
```

**关键约定**:
- Normalizer 在**全量数据** (train+val) 上拟合，不做 train/val 隔离
- `last_n_dims=1` 表示逐元素归一化（19 维 joint_state 各自独立缩放）
- 低方差维度 (range < 2e-2) 自动平移到零均值且不做缩放（参考 Diffusion Policy 附录建议），避免数值放大。相比旧版的 strict degeneracy 检测（1e-4），新阈值覆盖了实际中可能出现噪声放大的"低方差但非严格退化"维度。

### 使用方式

```python
# Agent 端
agent.load_normalizer_from_dataset(normalizer)
  → normalizer['action'] = SingleFieldLinearNormalizer
  → normalizer['joint_state'] = SingleFieldLinearNormalizer

# 训练: normalize
batch['obs'] = normalizer.normalize(batch['obs'])    # 仅归一化 obs 中有 normalizer 的 key
nactions = normalizer['action'].normalize(action)

# 推理: normalize → predict → unnormalize
cond = agent._build_cond(obs_dict)                    # 内含 normalize
pred = agent.predict_action_from_cond(cond)
action = normalizer['action'].unnormalize(pred)       # 还原到原始动作空间
```

### SingleFieldLinearNormalizer 特殊用法

```python
# 恒等 normalizer (camera 内外参)
normalizer['camera_intrinsic'] = SingleFieldLinearNormalizer.create_identity()
# scale=1, offset=0 → normalize/unnormalize 为恒等变换, 仅做 dtype 转换
```

## 数据增强框架

### 基类设计

`datasets/augmentation/base.py`:

```python
class Aug:
    def __init__(self, prob=1.0):
        self.prob = prob

    def __call__(self, x):
        if np.random.random() > self.prob:  # 按概率跳过
            return x
        return self._augment(x)             # 子类实现具体增强逻辑
```

**核心设计原则**:
- 所有增强在 **numpy 层面** 完成，在 `torch.from_numpy` 之前
- 概率控制在 `Aug.__call__` 中统一处理
- 增强是否生效完全由 `prob` 和是否在配置中声明该增强块决定；不在 `Aug.__init__` 签名中的额外字段（如 `enabled`）会触发 TypeError，不应传入

### 增强执行链

`BaseDataset.apply_augmentation()` (`base_dataset.py:98`):

```python
def apply_augmentation(self, data):
    if self.augmentation_cfg is None:    # 验证集: augmentation_cfg = None
        return data
    for modality, augs in self.augmentors.items():
        if modality not in data['obs'] or augs is None:
            continue
        for aug in augs:                   # 顺序执行: color → spatial → dropout
            data['obs'][modality] = aug(data['obs'][modality'])
    return data
```

**增强执行时机**: `BaseDataset.__getitem__` 中，`numpy → torch` 转换之前。

**验证集**: `get_validation_dataset()` 返回的验证集 `augmentation_cfg=None`，不执行任何增强。

### MultiTaskDataset 中的增强

`MultiTaskDataset` 自身不实现增强逻辑，而是将 `augmentation_cfg` 下发到各子数据集的 `_build_augmentors()`：

```python
# multi_task_dataset.py:36-40
if augmentation_cfg is not None:
    for dataset in self.datasets:
        if hasattr(dataset, '_build_augmentors'):
            dataset.augmentation_cfg = augmentation_cfg
            dataset._build_augmentors()
```

## 数据增强各模态实现

### RGB 增强 — RGBAug

`datasets/augmentation/rgb.py:8` — 基于 torchvision 的 ColorJitter。

| 参数 | 默认值 | 说明 |
|---|---|---|
| `brightness` | 0.3 | 亮度抖动范围 [1-b, 1+b] |
| `contrast` | 0.3 | 对比度范围 |
| `saturation` | 0.3 | 饱和度范围 |
| `hue` | 0.05 | 色相范围 [−h, +h] |
| `prob` | 1.0 | 执行概率 |

**时序一致性**: 同一序列的 T 帧共享相同的 jitter 参数 (`ColorJitter.get_params` 只调用一次)，避免帧间颜色闪烁。

**参考**: Diffusion Policy (RSS 2023) 的色彩增强策略。

### 点云增强体系

点云增强定义在 `datasets/augmentation/__init__.py` 中，通过 `PC_AUG_CLASSES` 注册表管理：

```python
PC_AUG_CLASSES = {
    'color':   PCColorJitter,   # HSV 颜色抖动
    'spatial': PCSpatialAug,    # Z 轴旋转 + XY 平移 + 缩放
    'dropout': PCDropout,       # 随机丢点
}
```

启用方式和执行顺序由配置中的 key 声明顺序决定。

#### PCColorJitter — HSV 颜色抖动

`datasets/augmentation/pc_color.py:8` — 在 HSV 空间对点云 RGB 通道做抖动。

| 参数 | 默认值 | 说明 |
|---|---|---|
| `brightness` | 0.2 | 亮度抖动 (factor∈[0.8, 1.2]) |
| `contrast` | 0.2 | 对比度抖动 |
| `saturation` | 0.3 | 饱和度抖动 |
| `hue` | 0.05 | 色相偏移 (绝对值) |
| `prob` | 1.0 | 执行概率 |

**处理流程**: brightness → contrast → saturation → hue，最后 clip 到 [0, 1]。

**适配性**: 仅当点云通道数 C ≥ 6 时生效（RGB 在 `[..., -3:]`，即最后 3 通道）。若 C=3（仅 XYZ 坐标），直接返回不做增强。兼容 pc_dim=6 (XYZ+RGB)、pc_dim=9 (XYZ+Normal+RGB) 等格式。

**参考**: ManiFlow (CoRL 2025) 的 `PointCloudColorJitter`，移植到 numpy 实现。

#### PCSpatialAug — 空间变换

`datasets/augmentation/pc_spatial.py:5` — Z 轴旋转 + XY 平面平移 + 均匀缩放。

| 参数 | 默认值 | 说明 |
|---|---|---|
| `rot_z` | 15.0° | Z 轴 (重力方向) 旋转角度范围 [−r, +r] |
| `trans_xy` | 0.10 | XY 平移范围 [−t, +t] (归一化单位) |
| `scale` | 1.02 | 缩放因子范围 [1/s, s] |
| `prob` | 1.0 | 执行概率 |

**变换细节**:
1. 随机采样 angle (度→弧度), dx, dy, s
2. 对 XYZ 坐标 (channel `[..., :3]`) 做二维旋转 (绕 Z 轴) + 缩放 + 平移
3. Z 坐标仅做缩放，不做平移
4. 所有 T 帧共享同一组变换参数

**参考**: RISE (IROS 2024) 的空间增强策略。

> **注意**: XYZ 通道受旋转/平移/缩放变换。当 `pc_dim >= 6` 时，通道 3:6（法向）自动同步 Z 轴旋转（不含平移/缩放）。颜色通道由 `PCColorJitter` 独立处理。当前所有配置使用 `pc_dim=3`，法向旋转逻辑不触发。

#### PCDropout — 随机丢点

`datasets/augmentation/pc_dropout.py:5` — 模拟灵巧手操作中的遮挡场景。

| 参数 | 默认值 | 说明 |
|---|---|---|
| `dropout_ratio` | 0.1 | 丢弃比例 |
| `prob` | 1.0 | 执行概率 |

被丢弃的点全部置零。所有 T 帧共享同一个 dropout mask。

### State 增强 — StateNoiseAug

`datasets/augmentation/state.py:5` — 对小维度本体感知状态加高斯噪声。

| 参数 | 默认值 | 说明 |
|---|---|---|
| `noise_std` | 0.005 | 噪声标准差 |
| `prob` | 1.0 | 执行概率 |

**适用性**: noise_std=0.005 相对于 limits 归一化后的 [-1,1] 范围是 0.25% 的扰动，非常轻微。

### 增强策略对比总结

| 增强 | 模态 | 时序一致 | 关键参数 | 参考论文 |
|---|---|---|---|---|
| RGBAug | RGB | ✓ (共享参数) | brightness/contrast/saturation/hue 0.3/0.3/0.3/0.05 | Diffusion Policy (RSS 2023) |
| PCColorJitter | PC RGB (C≥6) | ✓ (共享参数) | brightness/contrast/saturation/hue 0.2/0.2/0.3/0.05 | ManiFlow (CoRL 2025) |
| PCSpatialAug | PC XYZ (:3) | ✓ (共享变换) | rot_z=15°, trans_xy=0.10, scale=1.02 | RISE (IROS 2024) |
| PCDropout | PC XYZ (:3) | ✓ (共享 mask) | dropout_ratio=0.1 | — |
| StateNoiseAug | joint_state | ✗ (逐帧独立) | noise_std=0.005 | 通用 RL/IL 实践 |

**时序一致性**: RGB 和点云增强对所有 T 帧使用完全相同的随机参数/掩码，保证同一序列内感知一致性。State 噪声逐帧独立采样（模拟真实传感器噪声）。

## 多任务数据集 — MultiTaskDataset

`datasets/multi_task_dataset.py:8` — 独立的 Dataset 实现，非 BaseDataset 子类。

### 架构

```
MultiTaskDataset
├── datasets: List[BaseDataset]    ← N 个单任务数据集实例
├── task_names: List[str]          ← 任务名称 (env 导入用)
├── task_texts: List[str]          ← 任务自然语言描述 (CLIP 编码用)
├── sampling_strategy              ← 采样策略
├── normalizer_mode                ← 归一化模式
└── __getitem__:
    ├── 按策略选择 (task_idx, local_idx)
    ├── sample = datasets[task_idx][local_idx]   ← 走 BaseDataset.__getitem__
    ├── sample['obs']['task_text'] = task_texts[task_idx]
    └── sample['obs']['task_name'] = task_names[task_idx]
```

### 采样策略

| 策略 | 行为 | task_weights 是否生效 |
|---|---|---|
| `proportional` | 按各任务数据量占比采样，数据多的任务被采样更多 | 忽略 |
| `balanced` | 各任务等概率采样，每 epoch 总样本量 = 所有任务数据量之和 | 忽略 |
| `weighted` | 按 `task_weights` 归一化后的概率采样 | 必须提供 |

**balanced/weighted 采样细节**:
1. `compute_target_counts()`: `target = round(prob_i * total_length)`，凑整误差随机分配
2. `generate_epoch_indices()`: 每个 epoch 重新 shuffle 各任务内部的 local indices
3. 最终打乱所有 (task_idx, local_idx) 对

### Epoch 感知采样

```python
# deterministic=False (训练):
#   每个 epoch 重新生成 epoch_indices，同一 idx 在不同 epoch 返回不同样本
dataset.set_epoch(epoch)  # 由 Trainer 在 epoch 开始时调用

# deterministic=True (验证):
#   固定采样序列，保证每次 eval 结果可复现
dataset = MultiTaskDataset(..., deterministic=True)
```

### 归一化模式

| 模式 | 行为 | 适用场景 |
|---|---|---|
| `shared` | 拼接所有任务的 joint_state + action，拟合一个共享 normalizer | 任务间动作/状态范围相似 |
| `per_task` | 每个任务独立拟合 normalizer | 任务间量纲/范围差异大 |

`shared` 模式下会自动为 `camera_intrinsic` 和 `camera_extrinsic` 添加 identity normalizer。

### MultiTaskDataset 与 MultiTaskAgent text cache 联动

`MultiTaskDataset` 的 `task_texts` 列表和 `MultiTaskAgent` 的 `task_texts` 参数共享同一份文本列表，二者之间存在隐式依赖：

```
multitask_dit.yaml:
  dataset.task_texts: ["pick apple", "place cube", "stack block"]
  agent.task_texts: ${dataset.task_texts}        ← Hydra 引用, 保证一致性

MultiTaskDataset:
  __getitem__ → sample['obs']['task_text'] = self.task_texts[task_idx]
  → 返回字符串, 如 "pick apple"

MultiTaskAgent:
  __init__ → init_text_cache(task_texts)
    → 预计算所有文本的 CLIP embedding → task_emb_table (frozen buffer)
  _build_cond → get_text_emb(task_texts)
    → 查表获取预计算 embedding → text_proj → 拼接到 cond
```

**关键约束**:
- `task_texts` 在两处必须**完全一致**（顺序 + 内容），否则 `MultiTaskDataset` 注入的字符串在 `MultiTaskAgent` 的 `task_to_idx` 中查不到
- 查表失败时 `get_text_emb()` 会退回实时 CLIP 编码（fallback 路径），但会显著降低训练速度
- 配置中用 `${dataset.task_texts}` (Hydra 引用) 确保一致性，**不建议手动复制粘贴**
- 新增/修改任务描述时，两个位置同步更新

**text cache 的生命周期**:
```
初始化: MultiTaskAgent.__init__ → init_text_cache()
  → unique texts → CLIP 编码 (frozen) → task_emb_table (buffer)
  → task_to_idx: {"pick apple": 0, "place cube": 1, ...}

训练: get_text_emb(["pick apple", ...])
  → task_to_idx["pick apple"] → 1 → task_emb_table[1] → text_proj

保存: get_extra_state()
  → {"task_texts": ["pick apple", "place cube", "stack block"]}
  → 随 checkpoint 一起序列化

加载: set_extra_state(state)
  → task_to_idx = {"pick apple": 0, ...}  (重建索引)
  → task_emb_table 已在 buffer 中, 无需重新计算
```

### 验证集

`get_validation_dataset()` 聚合各子数据集的验证集（跳过返回 `None` 的子集），返回一个新的 MultiTaskDataset（`deterministic=True`，保留 `task_weights` 以支持 `weighted` 策略）。若所有子数据集验证集为空，返回 `None`。

```python
# 1. 样本 shape 检查
sample = dataset[0]
print(f"obs keys: {list(sample['obs'].keys())}")

    print(f"  obs['{k}']: {v.shape}  {v.dtype}")
```

## 常见问题排查

### 1. `ValueError: Episode X is too short`

> 详见 [序列采样机制 — 滑动窗口算法](#滑动窗口算法)

```
ValueError: Episode 3 is too short: length=6, min_required=8
```

**原因**: 某 episode 帧数 < 8，不满足 `horizon - pad_before - pad_after` 的最小长度。

**排查**:
```python
lengths = np.diff(np.insert(episode_ends, 0, 0))
bad = np.where(lengths < 8)[0]
print(f"Too-short episodes: {bad.tolist()}, lengths: {lengths[bad].tolist()}")
```

**处理**: 从 Zarr 中移除过短的 episode，或在采集时确保每个 episode ≥ 8 帧。

### 2. `KeyError: 'rgb'` / `KeyError: 'point_cloud'`

> 详见 [Agent-Dataset 选配指南](#agent-dataset-选配指南)

```
KeyError: 'rgb'
```

**原因**: Dataset 返回的 `obs` dict 中不包含 Agent 需要的 key。常见于 Agent-Dataset 错配。

**排查**: 检查 `sensor_modalities` 配置是否包含对应 key，Agent 类型与 Dataset 类型是否匹配。

### 3. `pc_dim` 与 Zarr 数据不匹配

> 详见 [各策略对模态数据的 Shape 与 Dtype 要求](#各策略对模态数据的-shape-与-dtype-要求)

**现象**: 训练正常但性能差，或 `PCColorJitter` 不生效。

**原因**: `pc_dim=3` 但 Zarr 中 PC 为 C=6 → ObsEncoder 的 `use_coord_only` 仅取 XYZ，颜色信息被丢弃。反之 `pc_dim=6` 但 Zarr 中 PC 为 C=3 → 索引 `[..., 3:6]` 越界或读到垃圾数据。

**排查**:
```python
c = root['data']['point_cloud'].shape[-1]
assert c == cfg.agent.pc_dim, f"Zarr C={c} != config pc_dim={cfg.agent.pc_dim}"
```

### 4. MultiTask 训练时 balanced 策略退化

> 详见 [Epoch 感知采样](#epoch-感知采样)

**现象**: 不同 epoch 返回相同样本，或某个任务总是被过度采样。

**原因**: `set_epoch(epoch)` 未被调用 → `epoch_indices` 永不刷新 → 每个 epoch 采样相同。

**排查**: 确认 `Trainer.train_epoch()` 中是否调用了 `dataset.set_epoch(epoch)`。该调用在 `trainer.py` 中已完成，但自定义训练循环需手动添加。

### 5. Normalizer 产生 NaN/Inf

> 详见 [数据归一化 — 拟合流程](#拟合流程)

**现象**: `loss=NaN`，模型输出异常。

**原因**: 某维度的 min≈max（关节卡死或动作恒定），`scale = 2.0 / (max - min)` 产生极大值，放大微小噪声导致数值不稳定。

**说明**: `range_eps` 默认为 `2e-2`，range < 0.02 的维度会被自动处理（scale=1, offset=-mean），避免 Noise Amplification。相比旧版的 1e-4 阈值，新默认值更好地覆盖了实际中可能出问题的"低方差但非严格退化"维度。如需更保守的行为，可在调用 `fit(range_eps=...)` 时覆写。

**排查**:
```python
normalizer = dataset.get_normalizer()
stats = normalizer.get_input_stats()
for k, s in stats.items():
    range_ = s['max'] - s['min']
    tiny = range_.abs() < 1e-3
    if tiny.any():
        print(f"{k}: {tiny.sum()} dims with range < 1e-3")
```

**处理**: 若某维度确实是固定值（如不用的手指关节），可在数据预处理中过滤；否则可将 `range_eps` 提高到更宽松的值（如 5e-2）。

### 6. text cache 查表失败

> 详见 [MultiTaskDataset 与 MultiTaskAgent text cache 联动](#multitaskdataset-与-multitaskagent-text-cache-联动)

**现象**: 多任务训练初期速度正常，后期变慢。

**原因**: `MultiTaskDataset.task_texts` 与 `MultiTaskAgent.task_texts` 不一致 → 查表失败 → 退回实时 CLIP 编码。

**排查**: 比对两端文本列表:
```python
print("Dataset:", dataset.task_texts)
print("Agent:  ", agent.task_to_idx.keys())
assert dataset.task_texts == list(agent.task_to_idx.keys())
```

## 关键约定与注意事项

> 排查具体问题时，优先查看 [常见问题排查](#常见问题排查) 中对应的条目；Zarr 文件检查参考 [数据验证与调试方法](#数据验证与调试方法)。

1. **全量归一化**: Normalizer 在 train+val 全量数据上拟合，不做 train/val 隔离。这确保了归一化参数稳定，但意味着验证集信息泄露到归一化统计量中。

2. **数据增强默认禁用**: 所有配置文件中增强块均被注释，需手动取消注释启用。增强是否执行完全由 `prob` 和配置键是否存在决定。完整可用示例见 [完整增强配置示例](#完整增强配置示例)。

3. **验证集无增强**: `get_validation_dataset()` 返回的实例 `augmentation_cfg=None`，确保验证指标不受随机性影响。

4. **动作 float32 强制转换**: `ReplayBuffer.copy_from_store()` 中，若 action dtype 不是 `float32`，会被强制转换为 `float32`。

5. **pc_dim 与 Zarr 通道数必须匹配**: 配置中的 `pc_dim` 必须与 Zarr 中 `point_cloud` 的最后一维一致。不匹配时不会报错但会静默丢弃或误读数据。（排查见 [常见问题 #3](#3-pc_dim-与-zarr-数据不匹配)）

6. **PCSpatialAug 的 Z 轴旋转前提**: 假设 Z 轴为重力方向。适用于桌面级操作场景，不适合任意朝向的灵巧手操作。

7. **MultiTaskDataset 的 epoch 同步**: 训练时必须调用 `set_epoch(epoch)`，否则 balanced/weighted 策略的跨 epoch 随机性消失，等同于 deterministic 模式。（排查见 [常见问题 #4](#4-multitask-训练时-balanced-策略退化)）

8. **序列最小长度**: episode 长度 ≥ 8 帧才可采样 (16 − 1 − 7 = 8)。更短的 episode 会触发 `ValueError`。（排查见 [常见问题 #1](#1-valueerror-episode-x-is-too-short)）

9. **pad 参数自动计算**: 配置中 `pad_before: ${eval:'${n_obs_steps} - 1'}` 和 `pad_after: ${eval:'${n_action_steps} - 1'}`，随 n_obs_steps/n_action_steps 自动调整，通常无需手动修改。

10. **多任务权重忽略警告**: 当 `sampling_strategy != 'weighted'` 但提供了 `task_weights` 时，会发出 `UserWarning` 提示权重被忽略。

11. **text cache 一致性**: `MultiTaskDataset.task_texts` 与 `MultiTaskAgent.task_texts` 必须完全一致。推荐用 Hydra 引用 `${dataset.task_texts}` 而非手动复制。（排查见 [常见问题 #6](#6-text-cache-查表失败)）
