## 1. 整体架构

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                              Agent 类继承体系                                  │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  BaseAgent (agents/core/base.py)                                             │
│  ├── obs_encoder: 感知编码器 (PointCloud / RGB / State / Text)                │
│  ├── action_decoder: 动作解码器 (Diffusion / FlowMatch)                       │
│  ├── normalizer: LinearNormalizer (mode='limits', 缩放到 [-1,1])             │
│  └── compute_loss → predict_action                                           │
│      │                                                                       │
│      ├── UNetDiffusionAgent                                                  │
│      │   ├── backbone: ConditionalUnet1D (1D CNN U-Net + FiLM)               │
│      │   ├── action_decoder: Diffusion (DDPM + DDIM)                         │
│      │   │                                                                   │
│      │   ├── DPAgent (agents/core/dp.py)                                     │
│      │   │   └── obs_encoder: DPObsEncoder (RGB + StateMLP)                  │
│      │   │                                                                   │
│      │   ├── DP3Agent (agents/core/dp3.py)                                   │
│      │   │   └── obs_encoder: DP3ObsEncoder (PointCloud + StateMLP)          │
│      │   │                                                                   │
│      │   └── MoEAgent (agents/core/moe.py)                                   │
│      │       └── obs_encoder: MoEObsEncoder (PointCloud + StateMLP + MoE)    │
│      │                                                                       │
│      ├── DiTXFlowMatchAgent                                                  │
│      │   ├── backbone: DiTXFlowMatch (Transformer + Cross-Attn + AdaLN-zero) │
│      │   ├── action_decoder: FlowMatchWithConsistency                        │
│      │   │                                                                   │
│      │   └── ManiFlowAgent (agents/core/maniflow.py)                         │
│      │       └── obs_encoder: ManiFlowObsEncoder (PointPN + StateMLP)        │
│      │                                                                       │
│      └── MultiTaskAgent (agents/core/multi_task.py)                          │
│          ├── backbone: DiT_Diffusion (Transformer + AdaLN, self-attn only)   │
│          ├── action_decoder: Diffusion (DDPM + DDIM)                         │
│          └── obs_encoder: DPObsEncoder + CLIPTextEncoder + text_proj         │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

### 训练前向数据流

```
batch['obs'] (B, 16, *)                batch['action'] (B, 16, 19)
  │                                       │
  ▼                                       ▼
normalizer.normalize()                  normalizer['action'].normalize()
  │                                       │
  ▼                                       │
modality_dropout (可选, 仅训练时)          │
  │  per-sample: 同一 batch 样本的        │
  │  两个时序步共享 dropout 状态          │
  ▼                                       │
[:, :n_obs_steps].flatten(0,1)            │
  │  (B, 2, *) → (B×2, *)                │
  ▼                                       │
obs_encoder(obs) → (cond, aux)            │
  │                                       │
  ▼                                       ▼
action_decoder.compute_loss(cond, nactions, ema_backbone=...)
  │
  ▼
total_loss = action_loss + aux_loss
  │
  ▼
total_loss.backward() → grad_accum → clip_grad_norm_(1.0) → optimizer.step → scheduler.step → EMA
```

### 推理前向数据流

```
obs_dict (仿真环境, 无 T 维度)
  │
  ▼
_build_cond(obs_dict)                      # normalize → truncate → flatten → encoder
  │
  ▼
action_decoder.predict_action(cond, template=zeros(B, 16, 19))
  │  Diffusion:  DDIM 迭代去噪 (默认 10 步)
  │  FlowMatch:  Euler ODE 积分 (默认 10 步)
  ▼
normalizer['action'].unnormalize(pred)     # [-1,1] → 原始动作空间
  │
  ▼
control_action = pred[:, n_obs_steps-1 : n_obs_steps-1 + n_action_steps]   → (B, 8, 19)
```

### 1. DPAgent — Diffusion Policy (RGB)

**配置文件**: `configs/dp.yaml`
**参考论文**: Diffusion Policy (Chi et al., RSS 2023)
**核心文件**: `agents/core/dp.py`, `agents/core/base.py`

```
┌──────────────────────────────────────────────────────┐
│                     DPAgent                          │
├──────────────────────────────────────────────────────┤
│                                                      │
│  观测输入: RGB (3×224×224) + joint_state (19,)       │
│                                                      │
│  ┌─────────────────────────────────────────────┐     │
│  │ DPObsEncoder                                │     │
│  │                                             │     │
│  │  RGB ──► DINO/CLIP/SigLIP/ResNet ──► f_rgb │     │
│  │           (frozen/full tune)                │     │
│  │                                             │     │
│  │  joint_state ──► StateMLP(19→64) ──► f_st  │     │
│  │                                             │     │
│  │  concat → flatten (B, out_dim)             │     │
│  │  out_dim = backbone.out_dim + 64           │     │
│  └──────────────────────┬──────────────────────┘     │
│                         │ cond (B, out_dim×2)        │
│                         ▼                            │
│  ┌─────────────────────────────────────────────┐     │
│  │ ConditionalUnet1D (backbone)                │     │
│  │  input_dim=19, down_dims=[256,512,1024]     │     │
│  │  kernel=5, n_groups=8                      │     │
│  │  FiLM cond: cond → (scale, bias)            │     │
│  │  Down → Mid → Up → Final Conv              │     │
│  └──────────────────────┬──────────────────────┘     │
│                         │ noise / sample pred       │
│                         ▼                            │
│  ┌─────────────────────────────────────────────┐     │
│  │ Diffusion (DDPM + DDIM scheduler)           │     │
│  │  train: noise ~ N(0,I) → add → denoise      │     │
│  │         pred vs target → MSE                │     │
│  │  infer: DDIM 10-step denoising              │     │
│  │  prediction_type: sample                    │     │
│  └─────────────────────────────────────────────┘     │
│                                                      │
│  核心特点:                                            │
│  - 2D 图像 + 预训练视觉 backbone 作为空间感知          │
│  - FiLM 条件 U-Net，与原始 Diffusion Policy 论文对齐   │
│  - RGB backbone 支持冻结/微调                          │
│  - 支持 modality_dropout（仅对归一化模态生效：joint_state）  │
│    注：action 虽在 normalizer 中注册，但不会进入 preprocess  │
│    门控逻辑，实际不可被 dropout                               │
└──────────────────────────────────────────────────────┘
```

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `rgb_backbone_name` | dino | resnet / clip / dino / siglip |
| `diffusion_step_embed_dim` | 256 | 扩散步编码维度 |
| `down_dims` | [256, 512, 1024] | U-Net 各层通道数 |
| `kernel_size` | 5 | 1D 卷积核大小 |
| `n_groups` | 8 | GroupNorm 分组数 |
| `condition_type` | film | film / mlp_film / cross_attention_film |
| `cond_predict_scale` | true | FiLM 条件注入预测 scale+bias (true) 或仅 bias (false) |
| `num_training_steps` | 100 | 扩散训练总步数 (T) |
| `num_inference_steps` | 10 | DDIM 推理步数 |
| `prediction_type` | sample | 扩散预测目标: sample (x0) / epsilon (noise) |

│  │                                             │     │

### 3. ManiFlowAgent — Flow Matching + Consistency

**配置文件**: `configs/maniflow.yaml`
**参考论文**: ManiFlow (Zhang et al., CoRL 2025)
**核心文件**: `agents/core/maniflow.py`, `agents/action_decoders/flowmatch.py`, `agents/action_decoders/backbone/ditx.py`

```
┌──────────────────────────────────────────────────────────────────┐
│                       ManiFlowAgent                              │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  观测输入: point_cloud (N×3) + joint_state (19,)                  │
│                                                                  │
│  ┌────────────────────────────────────────────────────────┐      │
│  │ ManiFlowObsEncoder                                     │      │
│  │                                                        │      │
│  │  point_cloud ──► FPS(↓1024) → Patch Tokenizer         │      │
│  │    (B×T,N,3)      patch_token: (K, C_pc)               │      │
│  │                   global_token: (1, C_pc)               │      │
│  │                                                        │      │
│  │  joint_state ──► StateMLP → expand → (K+1, 64)        │      │
│  │                                                        │      │
│  │  concat → tokens (B, T×(K+1), C_pc+64)                │      │
│  │  num_obs_tokens = (K+1) × 2, obs_token_dim = C_pc+64  │      │
│  └──────────────────────┬─────────────────────────────────┘      │
│                         │ cond: (B, N_tokens, token_dim)         │
│                         ▼                                        │
│  ┌────────────────────────────────────────────────────────┐      │
│  │ DiTXFlowMatch (backbone, hidden_dim=768, 12 layers)    │      │
│  │                                                        │      │
│  │  Action tokens: action_dim → embed + pos_embed        │      │
│  │  Obs tokens:    token_dim → embed + pos_embed         │      │
│  │  Time:    t + target_t → 各自 SinPosEmb → concat+fuse │      │
│  │                                                        │      │
│  │  DiTXBlock × 12:                                       │      │
│  │    Self-Attn (action ↔ action)  ─┐                    │      │
│  │    Cross-Attn (action → obs)    ─┤ AdaLN-zero 调制     │      │
│  │    MLP (4×hidden, GeLU)         ─┘                    │      │
│  │    门控: γ_sa, γ_ca, γ_mlp (初始化为 0)               │      │
│  │                                                        │      │
│  │  FinalLayer: RMSNorm → MLP → action_dim               │      │
│  └──────────────────────┬─────────────────────────────────┘      │
│                         │ v_θ(x, t, target_t, cond)              │
│                         ▼                                        │
│  ┌────────────────────────────────────────────────────────┐      │
│  │ FlowMatchWithConsistency                               │      │
│  │                                                        │      │
│  │  Flow Matching:                                        │      │
│  │    x_t = (1-t)·x_0 + t·x_1                             │      │
│  │    v_target = x_1 − x_0                                 │      │
│  │    L_flow = MSE(v_θ(x_t, t, 0, cond), v_target)       │      │
│  │                                                        │      │
│  │  Consistency Training (需 EMA teacher):                 │      │
│  │    teacher 从 x_{t+dt} 预估 x1 → 反推 v 在 x_t 处的值   │      │
│  │    L_cons = MSE(v_θ(x_t, t, dt, cond), v_teacher)     │      │
│  │                                                        │      │
│  │  total_loss = L_flow + L_cons                          │      │
│  │  batch 切分: flow_batchsize = max(1, min(B-1, ⌊B×r⌋))   │      │
│  │    consistency_batchsize = B - flow_batchsize           │      │
│  │    ema_model=None 时全量 flow（验证/无 EMA teacher）      │      │
│  │    B<2 或 consistency_batchsize=0 时仅 flow（降级）       │      │
│  │                                                        │      │
│  │  Inference: Euler 积分 dt=1/N, x_{t+dt}=x_t+v_θ·dt    │      │
│  └────────────────────────────────────────────────────────┘      │
│                                                                  │
│  核心特点:                                                        │
│  - Flow Matching: 学习速度场 v=x1-x0，比 Diffusion 采样路径更直     │
│  - Consistency Training: 使模型对 target_t∈(0,1] 都能给出正确速度   │
│  - DiTX Transformer: self-attn 建模 action token 间时序关系，       │
│    cross-attn 将 obs token 作为条件注入（非 concat）               │
│  - 训练时需传入 ema_backbone 以启用 consistency loss；              │
│    ema_backbone=None 时全量样本仅计算 flow loss（验证等场景）        │
│  - B<2 时 consistency_batchsize=0 降级为纯 flow 模式（防御性编程）   │
│  - DiTXBlock 9 路 modulation: self-attn/cross-attn/MLP 各 3 路      │
│    (shift, scale, gate)，gate 初始化为 0                          │
└──────────────────────────────────────────────────────────────────┘
```

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `encoder_type` | pointpn | pointpn / pointnext_tokenizer |
| `n_layers` | 12 | DiTX Block 层数 |
| `hidden_dim` | 768 | Transformer 隐藏维度 |
| `n_head` | 8 | 注意力头数 |
| `mlp_ratio` | 4.0 | MLP 隐藏层扩展比 |
| `p_drop_attn` | 0.1 | self-attn dropout |
| `qkv_bias` | true | Cross-Attn Q/K/V 线性层 bias |
| `qk_norm` | true | Cross-Attn Q/K LayerNorm |
| `pre_norm_modality` | false | 是否对 obs token 做 AdaLN-zero 预标准化 |
| `timestep_embed_dim` | 128 | t 正弦位置编码维度 |
| `target_t_embed_dim` | 128 | target_t 正弦位置编码维度 |
| `denoise_timesteps` | 10 | 推理时的 Euler 积分步数 |
| `flow_batch_ratio` | 0.75 | 批次中 flow loss 的样本比例 |
| `t_sample_mode_for_flow` | beta | flow 分支 t ~ Beta 分布 |
| `t_sample_mode_for_consistency` | discrete | consistency 分支 t 离散采样 |
| `dt_sample_mode_for_consistency` | uniform | consistency 分支 dt 采样模式 |
| `target_t_sample_mode` | relative | target_t = dt (relative) 或 t+dt (absolute) |

**FlowMatch vs Diffusion 对比**:

| 维度 | Diffusion (DP/DP3/MoE/MultiTask) | FlowMatch (ManiFlow) |
|---|---|---|
| 训练目标 | 预测噪声 ε 或干净样本 x₀ | 预测速度场 v = x₁ − x₀ |
| 前向过程 | x_t = √ᾱ_t·x₀ + √(1-ᾱ_t)·ε | x_t = (1−t)·x₀ + t·x₁ |
| 推理方式 | DDIM 迭代去噪 | Euler ODE 积分 |
| 损失函数 | MSE(noise_pred, noise) | MSE(v_pred, v_target) + L_cons |
| 采样路径 | 弯曲 (需 SDE→ODE 转换) | 直线 (设计上最优传输) |
| 额外需求 | 无 | EMA teacher (consistency 训练用) |

### 4. MoEAgent — Mixture of Experts DP3

**配置文件**: `configs/moe_dp3.yaml`
**参考论文**: MoE Diffusion Policy
**核心文件**: `agents/core/moe.py`, `agents/obs_encoder/plugins/moe.py`

```
┌──────────────────────────────────────────────────────────────────┐
│                         MoEAgent                                │
│                       继承自 UNetDiffusionAgent                   │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  观测输入: point_cloud (N×3) + joint_state (19,)                  │
│                                                                  │
│  ┌────────────────────────────────────────────────────────┐      │
│  │ MoEObsEncoder                                          │      │
│  │                                                        │      │
│  │  point_cloud ──► FPS → PC Encoder ──► feat_pc         │      │
│  │  joint_state ──► StateMLP ──► feat_state              │      │
│  │                                                        │      │
│  │  concat → z (B×T, in_dim)                             │      │
│  │             │                                          │      │
│  │             ▼                                          │      │
│  │  ┌──────────────────────────────────────┐             │      │
│  │  │ MoE Module                           │             │      │
│  │  │                                      │             │      │
│  │  │  Router: Linear(dim → n_experts)    │             │      │
│  │  │    → softmax → TopK(k=2)            │             │      │
│  │  │                                      │             │      │
│  │  │  ExpertMLP × N (默认 8)              │             │      │
│  │  │    每 expert: N 层 MLP(dim→hidden→out)│            │      │
│  │  │                                      │             │      │
│  │  │  output = Σ topk_prob_i · expert_i   │             │      │
│  │  │                                      │             │      │
│  │  │  Aux Loss:                           │             │      │
│  │  │    load_balance: N·Σ(f_i · p_i)     │             │      │
│  │  │    entropy: −mean(Σ p·log p)        │             │      │
│  │  │    λ=0.1, β=0.01                    │             │      │
│  │  └──────────────────────┬───────────────┘             │      │
│  │                         │ feat (B×T, out_dim)         │      │
│  └─────────────────────────┬─────────────────────────────┘      │
│                            │ cond (B, out_dim×2)                │
│                            ▼                                     │
│         ConditionalUnet1D + Diffusion                            │
│         (与 DP3Agent 共享 UNetDiffusionAgent)                     │
│                                                                  │
│  核心特点:                                                       │
│  - DP3 编码器后插入 Top-K Sparse MoE，条件化路由                   │
│  - 每次仅激活 2/8 experts，推理计算量不随 expert 数线性增长          │
│  - Aux loss 全程生效 (无 warmup)                                 │
│  - 推理时支持 override_idx: 手动指定 expert，跳过 router           │
│    (可用于消融实验: 验证 expert 是否学到了不同技能)                 │
│  - total_loss = diffusion_loss + moe_aux_loss                   │
└──────────────────────────────────────────────────────────────────┘
```

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `num_experts` | 8 | Expert 总数 |
| `top_k` | 2 | 每次激活的 Expert 数 |
| `moe_hidden_dim` | 256 | Expert MLP 隐藏层维度 |
| `moe_out_dim` | pc_out_dim + state_out_dim | MoE 输出维度 (自动计算) |
| `moe_num_layers` | 2 | 每个 Expert 的 MLP 层数 |
| `lambda_load` | 0.1 | Load Balancing loss 权重 |
| `beta_entropy` | 0.01 | Entropy loss 权重 |

```
┌──────────────────────────────────────────────────────────────────┐
│                      MultiTaskAgent                             │
│                      继承自 BaseAgent                             │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  观测输入: RGB (3×224×224) + joint_state (19,) + task_text (str) │
│                                                                  │
│  ┌─────────────────────────────────────┐                         │

│  ┌───────────────┴─────────────────────┐                         │

│  │                                     │                         │
│  │  task_text → CLIP(frozen) → emb    │                         │
│  │    │                                │                         │
│  │    ├─ text cache (可选):             │                         │
│  │    │   预计算所有 task 的 CLIP 向量    │                         │
│  │    │   推理时 O(1) 查表              │                         │
│  │    │                                │                         │
│  │    └─ text_proj (Linear, 可训练)     │                         │
│  │       emb → text_cond: (B, n_emb)  │                         │
│  └───────────────┬─────────────────────┘                         │
│                  │                                               │
│                  ▼                                               │
│  cond = concat[vis_cond, text_cond]  (B, vis_dim + n_emb)       │
│                  │                                               │
│                  ▼                                               │
│  ┌────────────────────────────────────────────────────────┐      │
│  │ DiT_Diffusion (backbone, n_emb=512, 6 layers)         │      │
│  │                                                        │      │
│  │  Action tokens: action → embed + learnable pos_emb    │      │
│  │  Cond: Linear(cond_dim → n_emb)                       │      │
│  │  Time: Sinusoidal → MLP → n_emb                       │      │
│  │  c = t_emb + cond_emb  (additive)                     │      │
│  │                                                        │      │
│  │  DiTBlock × 6:                                         │      │
│  │    Self-Attention + MLP                                │      │
│  │    AdaLN modulation → 6×hidden (shift,scale,gate ×2)  │      │
│  │    gate 初始化为 0                                     │      │
│  │                                                        │      │
│  │  FinalLayer: LayerNorm → AdaLN → Linear → action_dim  │      │
│  └──────────────────────┬─────────────────────────────────┘      │
│                         │                                        │
│                         ▼                                        │
│               Diffusion (DDPM + DDIM)                            │
│                                                                  │
│  核心特点:                                                       │
│  - 唯一支持自然语言任务指令的 Agent                                │
│  - CLIP text encoder 显式冻结 (requires_grad=False)，仅 text_proj 可训练│
│  - 文本缓存: 训练前预计算所有唯一 task_text 的 CLIP embedding，     │
│    推理时按字符串查表，避免重复编码                                │
│  - DiT backbone: pure self-attention，视觉和文本条件通过 concat   │
│    后经 cond_embedder 融合                                       │
│  - checkpoint 中保留 task_to_idx 映射 (get/set_extra_state)      │
│  - 数据由 MultiTaskDataset 注入 task_text / task_name 字段        │
└──────────────────────────────────────────────────────────────────┘
```

## 选型指南

| 场景 | 推荐 Agent | 原因 |
|---|---|---|
| 仅 RGB 相机、无深度/点云 | DPAgent | DP 专门为 RGB 设计，可利用预训练视觉 backbone |
| 有深度相机/点云、单任务 | DP3Agent | 点云提供 3D 几何信息，比 RGB 更直接，训练更稳定 |
| 有深度相机、追求更快采样 | ManiFlowAgent | Flow Matching 采样路径更直，理论可用更少步数 |
| 多技能/多物体、无语言标注 | MoEAgent | Expert 自动分化不同技能，无需人工标注 |
| 多任务、有自然语言指令 | MultiTaskAgent | 唯一支持文本条件，CLIP 文本编码提供语义泛化 |
| 快速原型/基线 | DP3Agent | 结构最简单，超参最少，训练最稳定 |

**选择优先级建议**:

1. 先确定感知模态: 只有 RGB → DPAgent; 有深度/点云 → DP3 系列
2. 再确定任务粒度: 单任务 → DP3Agent; 多技能 → MoEAgent; 多任务+语言 → MultiTaskAgent
3. 最后考虑精度/效率取舍: 追求效率 → ManiFlowAgent; 追求稳定 → DP3Agent

## 条件注入机制对比

三种条件注入方式的技术差异:

| 机制 | 使用位置 | 原理 | 计算形式 |
|---|---|---|---|
| **FiLM** | UNet1D (DP/DP3/MoE) | cond → Linear → (γ, β)，对 feature map 逐通道缩放+偏移 | `y = γ ⊙ x + β` |
| **AdaLN** | DiT (MultiTask) | cond → Linear → (shift, scale, gate) × 2 子层 | `y = (1+γ) ⊙ LN(x) + β` → gate ⊙ attn/mlp |
| **Cross-Attn** | DiTX (ManiFlow) | obs token 作为 K/V，action token 作为 Q，显式 token 间交互 | `y = softmax(QKᵀ/√d) · V` |

**差异解读**:

- **FiLM**: 计算最轻，适合 CNN backbone。cond 被编码为全局调制信号，逐通道作用于整个时序维度。缺点是无法建模 obs 与 action 之间的局部/时序交互。
- **AdaLN**: 适合 Transformer。在 LayerNorm 之后通过可学习的 shift/scale 注入条件，gate 初始化为 0 保证训练初期稳定性。条件通过加性方式 (t_emb + cond_emb) 融合。
  - `AdaLNZero`（ManiFlow `context_norm`）要求 `weight=0, bias[:dim]=1, bias[dim:]=0`（identity 初始化），`initialize_weights()` 中 `self.apply(init_fn)` 会覆盖此初始化，因此需在 `apply` 后显式调用 `self.context_norm.initialize_weights()` 恢复。
- **Cross-Attn**: 表达力最强。obs token 保留空间/局部结构（如 K 个 patch token），action token 可以有选择地关注不同空间位置的条件信息。代价是计算量更大 (O(N_action × N_obs))。

**条件维度差异**:

```
FiLM (DP/DP3/MoE):
  obs (B×2, *) → encoder → flat (B, concat_dim) → FiLM 注入 UNet1D
  cond_dim = out_dim × n_obs_steps (如 concat_dim=conc_dim)

AdaLN (MultiTask):
  obs (B×2, *) → encoder → flat (B, vis_dim)
  text → CLIP → text_proj → (B, n_emb)
  cond = concat[vis, text] → Linear → n_emb → AdaLN 注入 DiT

Cross-Attn (ManiFlow):
  obs (B×2, K+1, *) → encoder → tokens (B, N_tokens, token_dim)
  action (B, 16, 19) → embed → tokens (B, 16, hidden_dim)
  obs tokens 通过 Cross-Attn 作为 K/V 参与每层计算
```

## 感知编码器深度对比

### RGB Backbone 对比

| Backbone | 模型 | 参数量 | 输出维度 | 默认训练模式 | 适用场景 |
|---|---|---|---|---|---|
| resnet | ResNet-18 | ~11M | 512 | full (全微调) | 轻量级, 快速实验 |
| clip | ViT-B/32 | ~86M | 512 | freeze (冻结) | 视觉-语言对齐 |
| dino | DINOv2-B | ~86M | 768 | freeze (冻结) | 自监督视觉特征 |
| siglip | SigLIP-B/16 | ~87M | 768 | freeze (冻结) | 更好的视觉-语言对齐 |

- ResNet 是唯一支持全微调的 backbone (CNN 结构，参数量小)
- DINO/CLIP/SigLIP 均为 ViT 架构，默认冻结以减少显存和防止过拟合
- ImageProcessor 根据 backbone preset 自动选择对应的 resize/crop/normalize 参数

### 点云 Encoder 对比

```
┌───────────────────────────────────────────────────────────────┐
│                    点云编码器分类                               │
├───────────────────────────────────────────────────────────────┤
│                                                               │
│  全局编码器 (Global Encoder) ─ 用于 DP3/MoE                    │
│  ┌─────────────────────────────────────────────────────┐      │
│  │ PointNet (dp3):                                     │      │
│  │   3层 MLP [64,128,256] → max pooling → Linear+LN   │      │
│  │   最简单, 对点顺序不敏感, 输出 1 个 global token       │      │
│  │                                                     │      │
│  │ MultiStagePointNet (idp3):                          │      │
│  │   多级 PointNet + 层级聚合                           │      │
│  │   保留多尺度几何特征, 比单级 PointNet 更强              │      │
│  │                                                     │      │
│  │ PointNextEncoder (pointnext):                       │      │
│  │   多阶段 Set Abstraction (ball query + group)       │      │
│  │   感受野可控, 更适合复杂几何结构                       │      │
│  └─────────────────────────────────────────────────────┘      │
│                                                               │
│  Patch Tokenizer ─ 用于 ManiFlow                              │
│  ┌─────────────────────────────────────────────────────┐      │
│  │ PointPNTokenizer (pointpn):                         │      │
│  │   多阶段: FPS + LocalGrouper + LGA + ChannelExp     │      │
│  │   输出: K 个 patch token + 1 个 global token          │      │
│  │   每个 patch 保留局部几何信息                          │      │
│  │                                                     │      │
│  │ PointNextPatchTokenizer (pointnext_tokenizer):      │      │
│  │   Stem MLP → FPS → BallQuery → 局部 token            │      │
│  │   输出: K 个 patch token + 1 个 global token          │      │
│  └─────────────────────────────────────────────────────┘      │
│                                                               │
│  关键差异:                                                     │
│  - Global Encoder: 全盘点云 → 1 个向量, 压缩率高, 丢空间细节     │
│  - Patch Tokenizer: 全盘点云 → K+1 个向量, 保留局部结构          │
│    K 由 tokenizer 配置决定 (pointpn 约 64 个 patch)             │
│  - Patch Tokenizer 输出适合 Transformer 的 token 序列格式        │
│    (这正是 ManiFlow 选择 patch tokenizer 的原因)               │
└───────────────────────────────────────────────────────────────┘
```

| 编码器 | 类型 | 输出 | 使用 Agent | 特点 |
|---|---|---|---|---|
| PointNet (dp3) | Global | 1 token | DP3, MoE | 最简, 快 |
| MultiStagePointNet (idp3) | Global | 1 token | DP3, MoE | 多尺度, 默认 |
| PointNextEncoder (pointnext) | Global | 1 token | DP3, MoE | SA 层, 强 |
| PointPNTokenizer (pointpn) | Patch | K+1 tokens | ManiFlow | 多阶段 LG, 默认 |
| PointNextPatchTokenizer | Patch | K+1 tokens | ManiFlow | Stem+SA patch |

## 9. 架构对比总结

| 维度 | DPAgent | DP3Agent | ManiFlowAgent | MoEAgent | MultiTaskAgent |
|---|---|---|---|---|---|
| **感知模态** | RGB | PointCloud | PointCloud | PointCloud | RGB + Text |
| **Encoder** | DPObsEncoder | DP3ObsEncoder | ManiFlowObsEncoder | MoEObsEncoder | DPObsEncoder + CLIP Text |
| **Encoder 输出** | flat vector | flat vector | token sequence | MoE → flat | flat + text_emb |
| **Backbone** | UNet1D (CNN) | UNet1D (CNN) | DiTX (Transformer) | UNet1D (CNN) | DiT (Transformer) |
| **条件注入** | FiLM | FiLM | Cross-Attn | FiLM | AdaLN |
| **Action Decoder** | Diffusion | Diffusion | FlowMatch+Cons | Diffusion | Diffusion |
| **推理** | DDIM 10步 | DDIM 10步 | ODE 10步 | DDIM 10步 | DDIM 10步 |
| **特殊机制** | modality dropout | FPS | Consistency | Top-K MoE | Text Cache |
| **Aux Loss** | 无 | 无 | consistency | load_bal + entropy | 无 |
| **参数量级** | 中 | 中 | 大 (12L×768) | 大 (8 experts) | 中 (6L×512) |
| **Batch Size** | 256 | 256 | 128 | 256 | 64 |

### 继承层次与代码复用

```
BaseAgent                          ← preprocess, compute_loss, predict_action, normalizer
├── UNetDiffusionAgent             ← 装配 ConditionalUnet1D + Diffusion
│   ├── DPAgent                    ← 装配 DPObsEncoder
│   ├── DP3Agent                   ← 装配 DP3ObsEncoder
│   └── MoEAgent                   ← 装配 MoEObsEncoder, override _build_cond & predict_action
├── DiTXFlowMatchAgent             ← 装配 DiTXFlowMatch + FlowMatchWithConsistency
│   └── ManiFlowAgent              ← 装配 ManiFlowObsEncoder
└── MultiTaskAgent                 ← 独立装配 DiT_Diffusion + Diffusion + 文本处理
```

### 核心不变约束

```
horizon = 16, n_obs_steps = 2, n_action_steps = 8, action_dim = 19
```

- Input batch: `obs (B, 16, *)`, `action (B, 16, 19)`
- Preprocessed obs: `(B×2, *)` — 截取最近 2 步后展平 batch+time
- 推理输出: `control_action (B, 8, 19)` — 截取 8 步执行
- 归一化: `LinearNormalizer(mode='limits')`, 缩放到 `[-1, 1]`
- 优化器: `AdamW`, betas 见超参表
