### 1.5 `_randomize_table_height()` 实现

**文件**: `dexmani_sim/envs/base_env.py:513-521`

```python
def _randomize_table_height(self):
    self._table_height_offset = 0.0
    if not self.randomization.randomize_table_height or self._table_actor is None:
        return
    off = self.np_random.uniform(*self.randomization.table_height_range)
    self._table_height_offset = float(off)
    new_z = self._base_table_z + off
    old_pose = self._table_actor.get_pose()
    self._table_actor.set_pose(sapien.Pose(p=[old_pose.p[0], old_pose.p[1], new_z], q=old_pose.q))
```

- `_table_height_offset` 始终被初始化（line 514：`= 0.0`），保证下游代码即使在不启用时也能安全读取。
- `_table_actor is None` 保护：若场景无名为 `"table"` 的 actor（异常场景），静默跳过。
- 仅移动 Z，保持 XY 和旋转不变。

### P3 — Pour 任务的 ball Z 依赖 pre-physics bbox

**文件**: `dexmani_sim/envs/pour.py:76-77`

```python
ball_z = self.cup.bbox_top_z - 0.015
self.ball.set_pose(sapien.Pose(p=np.array([cup_posx, cup_posy, ball_z])))
```

ball 的 Z 使用 cup 的 pre-physics bbox 计算。cup 被放置在默认 posz=0.0，桌面下移后 cup 悬浮。ball 被放置在相对悬浮 cup 的正确位置。物理稳定化时两者一起下落。**实际影响较小**（两者同落），但初始放置位置与最终稳定位置不同，增加了稳定化步数。

### 补充发现 — `compute_failed()` 中的绝对 Z 阈值

以下任务的失败检测使用绝对世界 Z 坐标，桌面下移后阈值语义偏移：

| 任务 | 文件:行 | 条件 | 影响 |
|------|---------|------|------|
| PickBottle | `pick_bottle.py:56` | `bottle_z < -0.05` | 桌面下移 3cm 后，瓶子从 z=0 掉到 z=-0.03，离 -0.05 阈值更近 |
| MultiGrasp | `multi_grasp.py:66` | `callbell_z < 0.02` | 同上，初始 Z 降低使失败阈值更易触发 |

注意：这两个阈值在物体使用默认 `posz=0.0` 时仍以自己的初始 Z 为基准（物体 z=0.0，桌面下移不影响物体初始 Z）。但物理稳定化后物体落在下移的桌面上，Z 坐标变低。这些阈值在训练数据采集时以默认桌面高度校准，桌面下移后语义变化。

### 补充发现 — Normalizer 分布偏移

训练数据的 normalizer 在默认桌面高度下拟合。桌面下移 3cm 后：
- **Point cloud** 中所有桌面上的点的 Z 坐标整体下移 3cm——超出 normalizer 见过的范围。
- **Joint state** 不受影响（机器人基座固定）。

这是域随机化的固有特性，但当前 `mode='limits'` 的 normalizer 在 `(-0.03, 0.0)` 范围内仍有覆盖余量（normalizer 拟合的是整个 replay buffer 的 min/max，包含物体在不同高度的状态），实际影响需实验验证。

## 5. 验证方法

### 5.1 冒烟测试

对每个任务，启用桌面高度随机化跑少量 episode：

```bash
# 对所有 7 个任务分别执行
for task in pick_apple_messy pick_bottle open_box stack_cups multi_grasp place_milk_box pour; do
    python -c "
from dexmani_sim.envs.${task} import $(echo ${task} | sed 's/_/ /g' | sed 's/\b\(.\)/\u\1/g' | sed 's/ //g')
env = $(echo ${task} | sed 's/_/ /g' | sed 's/\b\(.\)/\u\1/g' | sed 's/ //g')(randomize_table_height=True)
obs, info = env.reset(seed=0)
print(f'${task}: table_height_offset={info.get(\"table_height_offset\", \"N/A\")}')
for obj_name in ['apple','plate','bottle','box','top_cup','bottom_cup','callbell','ball','milk_box','displaystand','cup','bowl','coaster']:
    if hasattr(env, obj_name):
        obj = getattr(env, obj_name)
        z = obj.get_pose().p[2]
        print(f'  {obj_name} z={z:.4f}')
"
done
```

验证每个物体的初始 Z 坐标是否等于 `_base_table_z + _table_height_offset`（即随桌面移动）。

### 5.2 完整评测回归

```bash
# 对每个策略跑两组评测：不启用 vs 启用桌面随机化
for policy in dp3 maniflow; do
```
