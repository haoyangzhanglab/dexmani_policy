## 2. 配置系统（引用）

训练完全由 Hydra 配置驱动，支持 `${eval:'...'}` 数学插值、`defaults` 继承、CLI override。输出目录规则为 `experiments/{policy}/{task}/{timestamp}_{seed}/`。

> **详见**：[02-配置系统](02-配置系统.md) — Hydra 入口机制、插值系统、配置校验、策略对比、CLI override 语法。

### 策略训练超参速查

| 参数 | DP | DP3 | ManiFlow | MoE DP3 | MultiTask |
|---|---|---|---|---|---|
| **batch_size** | 256 | 256 | 128 | 256 | 64 |
| **lr** | 1e-4 | 1e-4 | 1e-4 | 1e-4 | 1e-4 |
| **weight_decay** | 1e-6 | 1e-6 | 1e-3 | 1e-6 | 1e-6 |
| **betas** | (0.95,0.999) | (0.95,0.999) | (0.9,0.95) | (0.95,0.999) | (0.95,0.999) |
| **eval_interval** | 250 | 250 | 200 | 250 | 250 |
| **use_ema_teacher** | false | false | true | false | false |

> ManiFlow 的 `weight_decay=1e-3`、`betas=(0.9,0.95)` 与 Transformer backbone 惯例一致，batch_size=128 因 DiTX 显存占用更大。

## 3. 训练入口与组件装配

### 3.1 三种入口

```
train.py ──────────────────────────────────────────────
  build_train_components()   ← 完整组件装配
  validate_config()          ← 配置校验
  Trainer.train()            ← 训练循环

train_multi_task.py ────────────────────────────────────
  复用 build_train_components() + validate_config()
  Trainer.train()            ← 与单卡相同的 Trainer

train_ddp.py ──────────────────────────────────────────
  mp.spawn(ddp_worker, nprocs=num_gpus)
    每个 worker 复用 build_dataset_and_normalizer / build_model_and_ema /
                build_optimizer_and_scheduler
    → DDPTrainer.train()     ← DDP 包装的 Trainer
```

### 3.2 装配流程

```
1. build_dataset_and_normalizer(cfg)
     → dataset (BaseDataset / MultiTaskDataset) + normalizer (LinearNormalizer, limits)

2. DataLoader(dataset) → train_loader + val_loader

3. build_model_and_ema(cfg, device, normalizer)
     → model + ema_model (deepcopy) + ema_updater

4. build_optimizer_and_scheduler(cfg, model, batches_per_epoch)
     → optimizer (AdamW, 参数分组) + scheduler (cosine warmup)

5. instantiate(cfg.workspace) → TrainWorkspace
     → CheckpointStore + TopKCheckpointTracker + MultiLogger

6. instantiate(cfg.env_runner) → env_runner (可选)
```

### 3.3 Optimizer 参数分组

```python
# BaseAgent.configure_optimizer (base.py:118-136)
action_groups = self.action_decoder.model.get_optim_groups(weight_decay)
# backbone 内部按 bias/weight 分组（get_optim_group_with_no_decay）
for g in action_groups:
    g['lr'] = lr

# obs_encoder 参数按 ndim <= 1 分拆 decay/no_decay
decay_params, no_decay_params = [], []
for p in self.obs_encoder.parameters():
    if p.requires_grad:
        if p.ndim <= 1:     # bias、LayerNorm weight → weight_decay=0.0
            no_decay_params.append(p)
        else:                # Linear/Conv weight → weight_decay=obs_wd
            decay_params.append(p)
groups = action_groups + [
    {'params': decay_params, 'weight_decay': obs_wd, 'lr': obs_lr},
    {'params': no_decay_params, 'weight_decay': 0.0, 'lr': obs_lr},
]
# obs_lr 可独立设为 0 冻结 encoder
return torch.optim.AdamW([g for g in groups if g['params']], lr=lr, betas=betas)
```

`MultiTaskAgent` 额外添加 `text_proj` 参数组（weight + bias 分离）。各 Agent 的完整参数分组见 [04-模型架构](04-模型架构.md)。

### 3.4 Scheduler 总步数

```python
optimizer_steps_per_epoch = (batches_per_epoch + grad_accum_steps - 1) // grad_accum_steps
total_steps = optimizer_steps_per_epoch * num_epochs
```

支持：`cosine`（diffusers，默认）、`cosine_annealing`（PyTorch）、`one_cycle`、`cosine_with_restarts`、`linear`、`constant`。

## 4. 训练循环

### 4.1 主循环

```python
# trainer.py:284-348
workspace.load_for_resume(model, ema, ema_updater, optimizer, scheduler, "latest")
# → 恢复 model/ema/opt/sched state → (global_step, start_epoch)

model.to(device); ema_model.to(device); ema_model.eval()
optimizer_to(optimizer, device); optimizer.zero_grad(set_to_none=True)

for epoch in range(start_epoch, num_epochs):
    model.train()
    on_epoch_start(epoch)                # dataset.set_epoch(epoch)
    accum_step = 0; optimizer.zero_grad(set_to_none=True)

    for batch in train_loader:
        train_one_step(batch)            # forward → backward → accum
        global_step += 1
        if step % log_interval_steps == 0:
            log train scalars

    flush_gradient_accumulation()
    model.eval()
    finish_epoch(epoch, global_step)     # val / eval / checkpoint
```

### 4.2 train_one_step — 单步训练

```python
def train_one_step(self, batch):
    batch = dict_apply(batch, lambda x: x.to(self.device, non_blocking=True))

    loss_kwargs = {'ema_backbone': self.ema_model.action_decoder.model} if self.use_ema_teacher_for_consistency else {}
    raw_loss, log_dict = self.model.compute_loss(batch, **loss_kwargs)

    if not torch.isfinite(raw_loss):
        debug_path = self._save_nan_debug(raw_loss)       # 自动保存 debug checkpoint
        raise RuntimeError(
            f"Non-finite loss at epoch={self.current_epoch}, step={self.global_step}: "
            f"raw_loss={raw_loss.item()}. Debug checkpoint saved to {debug_path}"
        )

    scaled_loss = raw_loss / self.grad_accum_steps
    scaled_loss.backward()

    self.accum_step += 1
    if self.accum_step % self.grad_accum_steps == 0:
        self.apply_gradient_step()
```

NaN 检测（三层防护）：
- Loss 层面：`train_one_step` 中 `torch.isfinite(loss)` 检查，非有限值时自动保存 debug checkpoint（含 model/optimizer/scheduler state）
- 梯度层面：`apply_gradient_step` 中 `clip_grad_norm_` 返回 `total_norm`，若非有限则 `optimizer.zero_grad()` 清空污染梯度后抛异常
- DDP 死锁防护：`DDPTrainer.train()` 每个 training step 后用 `dist.all_reduce(err_flag, MAX)` 同步错误状态，所有 rank 一致退出，防止单 rank NaN 导致集群死锁

### 4.3 apply_gradient_step — 参数更新

```python
def apply_gradient_step(self):
    total_norm = clip_grad_norm_(self.model.parameters(), max_norm=self.grad_clip_norm)
    if not torch.isfinite(total_norm):
        self.optimizer.zero_grad(set_to_none=True)
        raise RuntimeError(f"Non-finite gradient norm at epoch={...}, step={...}: total_norm={total_norm}")
    self.optimizer.step()
    self.scheduler.step()
    self.optimizer.zero_grad(set_to_none=True)

    if self.use_ema and self.ema_updater is not None:
        model = self.model.module if hasattr(self.model, 'module') else self.model
        self.ema_updater.step(model)
```

### 4.4 梯度累积

`grad_accum_steps > 1` 时，每个 batch 的 loss 除以累积步数后反向传播，N 步后统一更新参数。等效增大有效 batch size 而不增加显存。

`flush_gradient_accumulation()` 在 epoch 结束时处理未满 N 步的尾部梯度：按实际累积步数缩放后执行更新。

### 4.5 finish_epoch — Epoch 结束任务

```
plan_epoch_end_tasks(epoch):  # 使用 1-based epoch_idx = epoch + 1

  task              触发条件                                     备注
  ────────────────  ───────────────────────────────────────────  ─────────────────
  sample            (epoch+1) % sample_interval == 0 或 last     action MSE（EMA 模型）
  validate          (epoch+1) % val_interval == 0 或 last        val loss（与训练一致传入 ema_backbone）
  evaluate          env_eval 启用 且 (last 或 (epoch+1) % eval_interval == 0)  仿真评测（EMA 模型）
  save_checkpoint   last 或 (epoch+1) % checkpoint_interval == 0  topk 追踪
  save_latest       last 或 (epoch+1) % val_interval == 0        latest.pt symlink

  eval_model     = ema_model if use_ema else model      # eval/sample 用 EMA
  checkpoint_model = model                              # 保存用训练模型
  
  注：FlowMatch（use_ema_teacher_for_consistency=true）验证时 student 固定为训练模型（teacher 为 EMA），确保 student ≠ teacher 使 consistency loss 有意义；DP/DP3 验证时 env_eval 禁用则用 EMA（确保 TopK fallback 与推理权重一致），env_eval 启用则用训练模型。对齐 ManiFlow_Policy 原始实现
```

## 5. 训练生命周期

以 DP3 为例（1000 epochs，batch=256）：

```
Epoch 0-10     初始阶段
  train/loss: 快速下降（~1.0 → ~0.1）
  train/lr: warmup 升至 1e-4（前 500 optimizer steps）

Epoch 10-100   早期收敛
  train/loss: 继续下降，渐趋平稳（~0.01-0.05）
  val/loss: 与 train loss 接近（无增强噪声）

Epoch 100-500  中期训练
  train/loss: 缓慢下降或平台期
  eval@250: 首次仿真评测

Epoch 500-1000 后期精调
  train/lr: cosine 衰减接近 eta_min
  topk: 保留前三高 success_rate 的 checkpoint
```

### 监控要点

| 指标 | 健康信号 | 警告信号 |
|---|---|---|
| `train/loss` | 前 50 epoch 持续下降 | 不下降、剧烈震荡、跳变 |
| `val/loss` | 与 train loss 同趋势 | 持续上升（过拟合） |
| `train/lr` | 按 scheduler 曲线平滑变化 | 不变化（scheduler 未生效） |
| `eval/success_rate` | 逐步上升至收敛 | 长期为 0、突然下降 |
| `train/aux_load_balancing_loss` (MoE) | 稳定在低值 | 持续增大（expert 坍缩） |

### 时长估算

| 策略 | GPU | batch | ~每 epoch | ~总时间 |
|---|---|---|---|---|
| DP3 | RTX 4090 | 256 | ~3s | ~50min |
| DP | RTX 4090 | 256 | ~5s | ~80min |
| ManiFlow | RTX 4090 | 128 | ~15s | ~4h |
| MoE DP3 | RTX 4090 | 256 | ~8s | ~2h |
| MultiTask | RTX 4090 | 64 | ~20s | ~5.5h |

## 6. 损失计算（引用）

所有 Agent 共享 `BaseAgent.compute_loss()` 管道：

```
batch['obs'] → normalizer.normalize() → modality_dropout (per-sample)
  → truncate[:, :n_obs_steps] → flatten(0,1) → obs_encoder(obs) → cond

batch['action'] → normalizer['action'].normalize() → nactions

cond + nactions → action_decoder.compute_loss(cond, nactions, ema_backbone=...)
  [Diffusion]:  noise ~ N(0,I) → denoise backbone → MSE(pred, target)
  [FlowMatch]:  批次拆分 flow/consistency → 速度 MSE + consistency teacher
  [MoE]:        total_loss = action_loss + aux_loss (load_bal + entropy)
```

各解码器的完整损失公式和数据流见 [04-模型架构](04-模型架构.md)。

## 7. EMA 指数移动平均

### 7.1 幂律衰减

```python
decay = 1 - (1 + step / inv_gamma) ** -power
# 默认: power=2/3, inv_gamma=1.0, min=0.0, max=0.9999, update_after_step=0
# step=0    → decay=0.0      (完全复制)
# step=100  → decay≈0.953
# step=1000 → decay≈0.990
```

### 7.2 更新机制

```python
def step(self, new_model):
    self.decay = self.get_decay(self.optimization_step)

    ema_params = dict(self.averaged_model.named_parameters())
    ema_buffers = dict(self.averaged_model.named_buffers())

    for name, param in new_model.named_parameters():
        if not param.requires_grad:
            ema_param.copy_(param.to(dtype=ema_param.dtype).data)  # 冻结参数直接复制
        else:
            ema_param.mul_(self.decay).add_(
                param.data.to(dtype=ema_param.dtype), alpha=1 - self.decay)

    for name, buf in new_model.named_buffers():                     # 同步 buffer
        if ema_buffers.get(name) is not None:
            ema_buffers[name].copy_(buf.to(dtype=ema_buffers[name].dtype).data)

    self.optimization_step += 1
```

EMA 直接操作 unwrapped 模型参数（`apply_gradient_step` 中已 strip `module.` 前缀），因此与 DDP 自然兼容。

### 7.3 使用场景

| 场景 | 模型 | 说明 |
|---|---|---|
| 训练期 action_mse | EMA | `eval_model.compute_action_mse()` |
| 训练期仿真评测 | EMA | `evaluate(eval_model)` |
| 验证 loss | 与训练一致传入 ema_backbone | `validate(model, ema_backbone=...)` |
| 检查点保存 | 训练模型 + EMA | `model_state` + `ema_model_state` 同时保存 |
| 独立评测 | EMA | `workspace.load_for_inference(use_ema=True)` |
| FlowMatch consistency（训练时） | EMA teacher | `compute_loss(batch, ema_backbone=ema_model)` |

## 8. 检查点系统

### 8.1 格式

```python
# checkpoint_io.py — simple.v1 格式
{
    "state": {
        "epoch": int,
        "global_step": int,
        "monitor": {"test_mean_score": float},
        "train_params": {"n_obs_steps": ..., "n_action_steps": ..., "action_dim": ..., "horizon": ...},
    },
    "weights": {
        "model": OrderedDict,     # 训练模型 state_dict (unwrapped)
        "ema_model": OrderedDict, # EMA 模型 state_dict
        "ema_updater": dict,
        "optimizer": dict,
        "scheduler": dict,
    },
    "_format": "simple.v1",
    "_saved_at": timestamp
}
```

### 8.2 保存策略

```python
# 原子写入: .tmp → os.replace() → .pt
checkpoint_store.save(filename, checkpoint)

# 触发条件 (trainer.py:75-85, 使用 epoch_idx = epoch + 1)
save_checkpoint: is_last or (checkpoint_interval_epochs > 0 and epoch_idx % checkpoint_interval_epochs == 0)
save_latest:     is_last or (val_interval_epochs > 0 and epoch_idx % val_interval_epochs == 0)
# checkpoint_interval_epochs = eval_interval_epochs (有 env_runner)
#                           = val_interval_epochs  (无 env_runner)
```

`save_latest` 通过 `os.replace(tmp_symlink, latest.pt)` 原子更新符号链接。当 `save_latest=True` 但 `save_checkpoint=False`（仅验证不评测的 epoch），只更新 latest symlink 指向已有 checkpoint，不创建新的 .pt 文件。当 `test_mean_score` 可用但 checkpoint 被 topk 清理时，`latest.pt` fallback 到 `best_path()`，确保符号链接始终有效。

### 8.3 TopK 追踪

按 `test_mean_score` 保留前 K 个 checkpoint（默认 k=3）。`topk_manifest.json` 记录排序，超出 topk 的旧文件自动删除。同分优先保留新 epoch。

### 8.4 恢复训练

```python
next_step, next_epoch = workspace.load_for_resume(
    model, ema_model, ema_updater, optimizer, scheduler, "latest"
)
# → 加载 latest.pt → 恢复全部组件 state → (global_step, epoch+1)
```

### 8.5 跨 DDP/单卡兼容

`fix_state_dict()` 自动处理 `module.` 前缀。Checkpoint 始终以 unwrapped 格式保存。详见 [07-多卡训练](07-多卡训练.md) 第 6 节。

## 9. 学习率调度

| 类型 | 实现 | 关键参数 |
|---|---|---|
| `cosine` | diffusers `get_cosine_schedule_with_warmup` | `num_warmup_steps`, `num_training_steps` |
| `cosine_annealing` | PyTorch `CosineAnnealingLR` | `T_max=total_steps`, `eta_min=0`（无 warmup） |
| `one_cycle` | PyTorch `OneCycleLR` | `pct_start=0.3`, `div_factor=25` |
| `cosine_with_restarts` | diffusers | warmup + cosine cycles |
| `linear` | diffusers | warmup + linear decay |
| `constant` | diffusers | 恒定 lr |

调度器在每个 `optimizer.step()` 后立即步进（即每次梯度累积完成时）。所有配置均使用 `cosine`（diffusers，含 500 warmup steps）。

## 10. 日志与实验管理

### 日志架构

```
MultiLogger
  ├─ JsonlLogger → experiments/<exp>/metrics.jsonl  (行式 JSON)
  └─ WandbLogger → wandb (默认 mode=offline)
```

`to_log_scalars()` 将日志 dict 转为纯标量：单元素 tensor → 标量；1D tensor → `{key}/expert_N`；video key → 跳过（WandbLogger 单独转 `wandb.Video`）。

### 写入时机

```
每 log_interval_steps:  train/loss, train/lr, train/loss_flow, ...
每 epoch end:
  sample   → sample/action_mse_error
  validate → val/loss
  evaluate → eval/success_rate, eval/avg_steps, eval/*_video
```

### 输出目录

```
experiments/<policy>/<task>/<timestamp>_<seed>/
├── config.yaml                     # Hydra 完整配置
├── metrics.jsonl                   # 结构化日志
├── checkpoints/
│   ├── latest.pt                   # 符号链接 → 最新
```

## 12. 多任务训练

`train_multi_task.py` 完全复用 `train.py` 的 `build_train_components()` 和 `Trainer`，差异仅在配置：`cfg.dataset` → `MultiTaskDataset`，`cfg.agent` → `MultiTaskAgent`。

### Epoch 同步

```python
# trainer.py:278-282
def on_epoch_start(self, epoch):
    if hasattr(self.train_loader.dataset, 'set_epoch'):
        self.train_loader.dataset.set_epoch(epoch)
    if hasattr(self.model, 'set_epoch'):
        self.model.set_epoch(epoch)
```

`MultiTaskDataset.set_epoch(epoch)` 通过 `multiprocessing.Value` 共享内存将 epoch 同步到 DataLoader worker 进程。在 `persistent_workers=True` 下，worker fork 时继承 shared memory 引用，主进程写入 `_epoch_val.value` 后 worker 可即时感知变更，确保 `balanced`/`weighted` 策略每 epoch 正确刷新。

### text cache 联动

`MultiTaskDataset.task_texts` 通过 `${dataset.task_texts}` 引用传递给 `MultiTaskAgent.init_text_cache()`，预计算 CLIP embedding 存入 buffer。训练时 O(1) 查表。详见 [03-数据集与增强](03-数据集与增强.md) "text cache 联动"。

## 13. 多卡 DDP 训练（引用）

DDP 训练通过 `mp.spawn` 启动 N 个 worker，每个 rank 使用 `DistributedSampler` 做数据分片。rank 0 独享 logging/checkpoint/eval，其他 rank 通过 `CheckpointStore` 直接读 checkpoint（workspace=None）。DDP 在 `loss.backward()` 时自动 all-reduce 梯度。

## 14. 配置验证

`validate_config()` 在训练启动前执行：

| 校验类别 | 规则 |
|---|---|
| 窗口约束 | `n_obs_steps ≤ horizon`, `n_action_steps ≤ horizon`, `n_obs_steps-1+n_action_steps ≤ horizon` |
| 数据路径 | `zarr_path` 存在性校验（单任务 `cfg.dataset.zarr_path`，多任务 `cfg.dataset.datasets[i].zarr_path`），不存在则抛出 `FileNotFoundError` |
| 优化器 | `obs_lr ≥ 0`（0 表示冻结 encoder） |
| MultiTask | `task_texts` 长度匹配 `task_names`；`task_weights` 长度匹配 `task_names`（若存在）；env_runner `task_configs` 数量匹配 `task_names`；env_runner task_text 在 dataset.task_texts 中 |
| ManiFlow | `0 < flow_batch_ratio < 1`；`target_t_sample_mode ∈ {relative, absolute}`；`batch_size ≥ 2`（consistency training 需要至少 2 个样本做 flow/consistency split）；`use_ema=false` 或 `use_ema_teacher_for_consistency=false` 时发出 UserWarning（纯 flow 训练会导致推理条件分布偏移） |
| MoE | `top_k ≤ num_experts` |
| 增强一致性 | `pc_dim < 6` 时禁止 PC color augmentation |
| modality_dropout | 对非归一化模态（point_cloud, rgb）配置 `modality_dropout_probs > 0` 时发出 UserWarning（这些模态不走 normalizer，dropout 不生效） |
| DataLoader | `persistent_workers=True` 与 `num_workers=0` 不兼容（persistent workers 需至少 1 个 worker） |

## 15. FAQ

### Q1: 训练 loss 不下降

降低 lr 测试（`optimizer.lr=1e-5`）；检查 normalizer 是否缩放到 [-1,1]；确认 `obs_lr=0` 时 encoder 已有预训练权重。

### Q2: DDP 训练 hang 住

检查 MASTER_PORT 是否被占用；确认 GPU 数量匹配 `torch.cuda.device_count()`。

### Q3: 恢复训练后 loss 跳变

确认 `workspace.load_for_resume()` 传入了所有组件（model, ema, ema_updater, optimizer, scheduler），checkpoint 包含完整 optimizer/scheduler state。

### Q4: 多任务某 task loss 异常高

尝试 `sampling_strategy=weighted` + `task_weights=[2.0,1.0,1.0]`；检查 `normalizer_mode=shared` 下各 task 分布差异。

### Q5: OOM

减小 batch + 梯度累积：`'dataloader.batch_size=64' 'training.loop.grad_accum_steps=4'`；点云降采样 `agent.num_points=512`。

### Q6: success_rate=0 但 loss 正常

多 denoise 步数对比：`'eval.offline.denoise_timesteps_list=[5,10,20,50]'`；确认 EMA 权重正确更新。

### Q7: Val loss 低于 train loss

正常现象 — train 含增强噪声，val 无增强。若差距持续扩大，检查增强是否过强。

### Q8: FlowMatch consistency 训练 batch<2 警告

**现象**：训练时出现 "FlowMatch: batch_size=1 < 2, consistency training disabled for this batch"

**原因**：当 `grad_accum_steps > batch_size` 时，每个 micro-batch 的 batch_size < 2，consistency 训练降级为纯 flow 模式。

**解决方案**：
1. **配置校验已覆盖**：`validate_config()` 会在训练前检查 `batch_size >= 2`，拒绝不合理配置（每个 micro-batch 独立做 flow/consistency split，grad_accum_steps 不影响 batching）
2. **运行时降级**：如果绕过配置校验，运行时会降级为纯 flow 模式（仅计算 flow loss）
3. **建议配置**：增大 `dataloader.batch_size` 或减小 `training.loop.grad_accum_steps`

**注意**：此 warning 主要是防御性编程，正常训练不应频繁出现。

## 附录：关键约定

1. **Scheduler 总步数**：`ceil(batches_per_epoch / grad_accum_steps) * num_epochs`（向上取整），非 `batches_per_epoch * num_epochs`
2. **EMA checkpoint**：保存 unwrapped 训练模型 + EMA 两个 state_dict，恢复时按场景选择
3. **DDP seed**：两阶段设置 — 模型初始化前 `set_seed(seed)` 保证所有 rank 初始权重一致；模型构建后 `set_seed(seed + rank)` 增加各 rank 数据增强多样性
4. **Normalizer** rank 0 构建后 broadcast 同步到其他 rank
5. **`fix_state_dict`** 自动处理 `module.` 前缀，用户无需干预
6. **梯度累积**：尾部 `flush_gradient_accumulation()` 确保不浪费计算
7. **FlowMatch consistency** 依赖 `use_ema_teacher_for_consistency=true` + 传入 `ema_backbone`（训练和验证均传入，对齐 ManiFlow_Policy）；若未启用 EMA teacher，`validate_config()` 会发出 UserWarning（推理时 target_t>0 会发生条件分布偏移）
8. **Wandb 离线**：`mode: offline` 避免网络依赖，完成后 `wandb sync` 手动上传
9. **Workspace 关闭**：`atexit.register` + `SIGINT/SIGTERM` handler 双重保障，`close()` 幂等
10. **`torch.set_float32_matmul_precision('high')`**：训练开始时利用 TF32 加速
