# 09 — 域随机化机制

> 覆盖 `texture_random`、`instance_random` 便捷开关，以及 `randomize_table_height` 独立配置项。
> 横跨 `DexMani_Policy`（训练/评测编排层）与 `DexMani_Sim`（仿真环境层）两个仓库。

---

## 1. 架构概览

```
DexMani_Policy (编排层)
  configs/*.yaml  ──Hydra──▶  SimRunner(env_kwargs)
                                    │
                      _expand_env_kwargs()  ←── 便捷开关展开
                                    │
                          make_env() ──▶  TaskEnv(**expanded_kwargs)
                                                │
                                    DexMani_Sim (仿真层)
                                      resolve_env_kwargs()  ←── kwarg 路由
                                      RandomizationConfig dataclass
                                      BaseEnv.reset() 每 episode 应用随机化
```

两层分工：
- **DexMani_Policy** (`sim_runner.py`): 定义高层便捷开关，展开为个体 kwarg
- **DexMani_Sim** (`config.py` + `base_env.py`): 接收个体 kwarg，存储到 `RandomizationConfig`，在 `reset()` 中执行

## 2. 便捷开关机制

`SimRunner._expand_env_kwargs()` (`sim_runner.py:40-66`) 是连接两层的关键桥梁。

```python
# sim_runner.py:53-66
expanded = dict(env_kwargs)

if expanded.pop('texture_random', False):
    expanded.setdefault('use_texture_randomization', True)
    expanded.setdefault('per_episode_skybox', True)
    expanded.setdefault('randomize_light', True)
    expanded.setdefault('randomize_object_texture', True)
    expanded.setdefault('randomize_camera_pose', True)

if expanded.pop('instance_random', False):
    expanded.setdefault('randomize_model_id', True)

if expanded.pop('table_random', False):
    expanded.setdefault('randomize_table_height', True)
```

关键语义：

- **pop + setdefault**: 高层开关被移除（下游不可见），个体 kwarg 只在开关为 `true` 时才注入
- **setdefault 优先级**: 如果用户在 YAML 中已显式设置了同名个体 kwarg（如 `randomize_camera_pose: false`），`setdefault` 不会覆盖，显式值优先
- **展开后通过 `**env_kwargs` 传入环境构造函数** (`sim_runner.py:82`)

### 开关对照表

| 便捷开关 | 展开为 | 默认值 |
|---|---|---|
| `texture_random: true` | `use_texture_randomization` | 场景纹理随机（构建时） |
| | `per_episode_skybox` | 每 episode 换天空盒 |
| | `randomize_light` | 光照位置+颜色随机 |
| | `randomize_object_texture` | 物体纹理颜色+粗糙度随机 |
| | `randomize_camera_pose` | 相机位姿微扰 |
| `instance_random: true` | `randomize_model_id` | 模型变体随机 |
| `table_random: true` | `randomize_table_height` | 桌面高度随机 |
| *(无便捷开关)* | `crazy_light_rate` | 逐步光照闪烁 |

> **注意**: `crazy_light_rate` 没有对应的便捷开关，必须通过 CLI override 或 YAML 显式设置。
> `randomize_object_scale` 不存在于 DexMani_Sim 中（`RandomizationConfig` 无此字段），
> `_expand_env_kwargs` 中已移除该项（2026-06-23 修复）。

### 当前配置状态

所有 6 个 YAML config 中 `table_random` 和 `instance_random` 默认为 `true`，`texture_random` 默认为 `false`。
`crazy_light_rate` 不在任何 YAML 中（无便捷开关）。

## 3. 实例随机化 (`instance_random`)

### 3.1 完整链路

```
YAML config                          DexMani_Policy
  instance_random: false  ──────────▶  sim_runner.py:62-63
                                       ↓ pop + setdefault
                                       randomize_model_id=True
                                                       │
DexMani_Sim                                           │
  resolve_env_kwargs() ← config.py:123-163            │
  │ 路由到 RandomizationConfig                         │
  ▼                                                    │
  RandomizationConfig  ← config.py:59-83 ◀────────────┘
    randomize_model_id: bool = False
  │
  ▼
  BaseEnv._maybe_randomize_object()  ← base_env.py:599-613
    ├─ model_id 随机化分支 (randomize_model_id 且 pool 非空):
    │   ├─ 从 safe_pool（若提供）或 pool 中随机选 model_id
    │   ├─ 若 model_id 在 scale_overrides 中 → 使用覆盖缩放
    │   ├─ 否则若 base_scale 非空 → 显式重置为 base_scale（防状态泄漏）
    │   └─ scale_overrides/base_scale 是确定性映射，非随机化
    └─ 若 kwargs 为空 → return（不执行 reinit）
```

### 3.2 执行细节

**`_maybe_randomize_object(obj, pool, safe_pool, base_scale, *, on_reinit, scale_overrides)` (`base_env.py:599-613`):**

**model_id 随机化分支**（`randomize_model_id=True` 且 `pool` 非空）：

1. 优先从 `safe_pool` 选 model_id，若 `None` 则从 `pool` 选。使用 `self.np_random.choice()`
2. **scale_overrides**: 若选中的 model_id 在 dict 中 → `kwargs["user_scale"] = scale_overrides[model_id]`
3. **显式重置**: 若 model_id 不在 scale_overrides 中但 `base_scale` 非空 → `kwargs["user_scale"] = base_scale`。**这是关键修复**：`RoboTwinObject.reinit()` (`robotwin.py:165-179`) 在 `user_scale=None` 时保留 `self.user_scale` 旧值。若无此分支，上一 episode 的 scale_override model 会将其 `user_scale` 泄漏给本 episode 的无 override model（详见 §3.5）
4. **无独立 scale 随机化**: DexMani_Sim 的 `RandomizationConfig` 中不存在 `randomize_object_scale` 字段。当前 scale 变化仅作为 model_id 切换的附带效果（通过 `scale_overrides` map），或由各 task 在 `reset_objects()` 中自行实现（如 `stack_cups.py` 的自定义缩放逻辑）。策略侧 `_expand_env_kwargs` 中的 `randomize_object_scale` 已于 2026-06-23 移除（该参数从未在仿真侧实现，会导致 TypeError）

> **注意**: 独立的对象缩放随机化功能需要先在 DexMani_Sim 的 `RandomizationConfig` 中添加字段并在 `BaseEnv.reset()` 中实现，才能在策略侧通过便捷开关暴露。

> **注**: 仿真侧 `RandomizationConfig` 无 `randomize_object_scale` 字段。当前 `_maybe_randomize_object` 仅支持 model_id 随机化。独立的对象缩放随机化功能需要先在 `RandomizationConfig` 中添加字段并在 `BaseEnv.reset()` 中实现。

**reinit 语义**: 若 `kwargs` 非空，调用 `obj.reinit(**kwargs)` 销毁并重建 actor（保留原始 pose）。`on_reinit` 回调在 reinit 后执行（如 `pour.py` 用于重新应用杯子的 cmass 偏移）。

### 3.3 各任务处理

| 任务 | manipulated_objects | safe_pool | 特殊处理 |
|---|---|---|---|
| `pick_apple_messy` | apple, plate | apple 全量 (2/2), plate 全量 (1/1) | — |
| `pick_bottle` | bottle | 排除 `{1,2,3,5,8,12,17,21}` | 50-demo 验证 82%；21 为 physics unstable |
| `stack_cups` | top_cup, bottom_cup | top=[0,3], bottom=[1,3,5] | 50-demo 验证 82%(41/50)；窄缩放+bbox 嵌套约束（详见 §3.4） |
| `place_milk_box` | milk_box, displaystand | milk_box=[0,3], scale_overrides={0:1.0} | 50-demo 验证 84%(42/50)；variant 从 1→2；stand 全量 5/5 |
| `pour` | cup, bowl, coaster | cup 过滤 id>5 (models 6-12 为碗形不兼容) | cup_physics cmass 偏移 |
| `multi_grasp` | callbell, ball | callbell=[0,3,4,5] (排除 {1,2}) | 25-demo 验证 96%；ball 仅 1 variant |
| `open_box` | FlipBox | 不支持 `_maybe_randomize_object` | FlipBox 不是 RoboTwinObject |

> **safe_pool 设计原则**: (1) 保证 >80% mimic_gen 成功率；(2) safe_pool 内 variant 越多越好；(3) 多 variant 对象 model_id 随机化时 skip scale 随机化；(4) 单 variant 对象保留 scale 随机化作为唯一多样性来源。2026-06-22 验证：pick_bottle 82% + stack_cups 86% + place_milk_box 80% + pour 90% = 169/200 (84.5%)。

### 3.4 stack_cups 特殊逻辑

`stack_cups` (`stack_cups.py:43-91`) **完全绕过 `_maybe_randomize_object`**，使用自定义随机化逻辑（`base_env.py:584` 有相应注释标注）：

- **原因**: 需要 bbox 嵌套约束 + model pair 选择 + MIN_SCALE_RATIO，无法塞进通用 dispatch
- **safe_pool**: top=[0,3], bottom=[1,3,5]（5 对有效组合，top≠bottom 排除 (3,3)）
- **Scale 随机化原则**: stack_cups 自定义缩放逻辑独立于 `randomize_model_id` 控制。model_id 随机化时使用窄缩放范围 `(0.90, 1.10)`，反之使用全范围（2026-06-22 修复）
- **窄缩放范围**: `(0.90, 1.10)` — top 以 0.9 为基数，bottom 以 1.25 为基数
- **最小比例约束**: `bottom_scale ≥ 1.40 × top_scale`（保证顶部杯子能嵌套进底部杯子）
- **bbox 嵌套检查**: 按 model_id pairs 遍历，找满足 `top.bbox_size_xy < bottom.bbox_size_xy × 0.85` 的配对。若失败，迭代放大 bottom_scale 最多 4 次
- **兜底**: 若找不到合适配对，回退到 `model_id=(0,1)`, `top_scale=0.80, bottom_scale=1.30`
- **50-demo 验证**: combined mode (model_id + scale, 修复前): 82% (41/50)；修复后 model_id-only mode: 86% (43/50, +4pp) — 证实 skip scale 原则有效。top=0 达 94% (33/35)，top=3 为 67% (10/15) 是唯一 <80% 组件
- **排除项**: top=2 (json_scale=0.11, 73% — 最大杯型导致抓手闭合点不匹配), top=1 (60%), bottom=4 (0% — 不兼容几何)

### 3.5 State Leakage Bug 及修复

**bug 描述** (`base_env.py:591-594`, 2026-06-19 修复):

`RoboTwinObject.reinit()` 在 `user_scale=None` 时保留 `self.user_scale` 旧值（`robotwin.py:160-175`）。当 `_maybe_randomize_object` 仅为无 scale_override 的 model 设置 `kwargs={"model_id": N}` 时，reinit 继承上一 episode 的 `_user_scale`。

**复现** (以 `place_milk_box` 为例):
```
Episode 1: 选 model=0 → kwargs = {model_id: 0, user_scale: 1.0}  # override
           → reinit → self.user_scale = 1.0
Episode 2: 选 model=3 → kwargs = {model_id: 3}  # 无 user_scale！
           → reinit → self.user_scale 保持 1.0 ← 泄漏！
```
此时 model 3 的实际缩放为 1.0（应为 0.8），导致成功率从 ~88% 暴跌至 ~27%。

**修复**: 在 model_id 随机化分支中添加 `elif base_scale is not None: kwargs["user_scale"] = base_scale`。每次 reinit 调用均显式设置 `user_scale`，彻底切断跨 episode 状态泄漏。

**修复验证**: `place_milk_box` safe_pool=[0,3] + scale_overrides={0:1.0}，50-demo 验证 milk_box_id0=84% (16/19)、milk_box_id3=84% (26/31) — 两者独立成功率相等，完全解耦。

## 5. 桌面高度随机化 (`randomize_table_height`)

### 5.1 配置

```python
# config.py:62-63
randomize_table_height: bool = False
table_height_range: tuple[float, float] = (-0.05, 0.05)
```

- `RandomizationConfig` 默认关闭；策略 YAML 中 `table_random: true` 默认开启
- 偏移范围 `(-0.05, 0.05)`（默认）：桌子可在 ±5 cm 范围内移动

### 5.2 执行

```python
# base_env.py:516-527
def _randomize_table_height(self):
    self._table_height_offset = 0.0
    if not self.randomization.randomize_table_height:
        return
    for actor in self.scene.get_all_actors():
        if actor.get_name() == "table":
            off = self.np_random.uniform(*self.randomization.table_height_range)
            self._table_height_offset = float(off)
            new_z = self._base_table_z + off
            old_pose = actor.get_pose()
            actor.set_pose(sapien.Pose(p=[old_pose.p[0], old_pose.p[1], new_z], q=old_pose.q))
            return
```

关键点：

- `_base_table_z` 在 `BaseEnv.__init__()` 中从 table actor 的初始 Z 捕获（`base_env.py:87-92`），默认值 `-0.02`
- Table 在 `constructor.py:119` 以 `p=[0.4, 0, -0.02]` 创建，所以 `_base_table_z` 通常是 `-0.02`
- 随机化后 table Z ∈ `[-0.05, -0.02]`
- `_table_height_offset` 总是被设置（即使关闭时也设为 `0.0`），其他代码可安全使用 `getattr(self, '_table_height_offset', 0.0)`

### 5.3 调用时序

在 `reset()` (`base_env.py:672-715`) 中，`_randomize_table_height()` **最先调用**（line 676），早于：

1. robot reset
2. `reset_objects()`（物体放置 + 实例随机化）
3. stability check
4. `_apply_domain_randomization()`（摩擦/质量）
5. lighting / object texture / skybox / camera 随机化
6. `_collect_scene_info()` — 记录 `table_height_offset` 到 `scene_info`

### 5.4 消费者分析

**唯一使用 `_table_height_offset` 的任务**：`PlaceMilkBox` (`place_milk_box.py:60,88`)

```python
# 物体放置时
table_z = self._base_table_z + getattr(self, '_table_height_offset', 0.0)
self.milk_box.set_pose(pos_x=..., pos_y=..., posz=table_z + 0.08, ...)

# 检测失败时
box_z = self.milk_box.get_pose().p[2]
table_z = self._base_table_z + getattr(self, '_table_height_offset', 0.0)
if box_z < table_z + 0.02: ...
```

**不使用 `_table_height_offset` 的任务**（以固定世界坐标 `posz=0.0` 放置物体）：

| 任务 | 物体 | Z 坐标 |
|---|---|---|
| `pick_apple_messy` | apple, plate | 默认 `posz=0.0` |
| `pick_bottle` | bottle | 默认 `posz=0.0` |
| `stack_cups` | top_cup, bottom_cup | 默认 `posz=0.0` |
| `pour` | cup, ball, bowl, coaster | 默认 `posz=0.0`（ball 用 `cup.bbox_top_z-0.015`） |
| `multi_grasp` | callbell, ball | 默认 `posz=0.0` |
| `open_box` | FlipBox | 默认 `posz=0.0` |

**实际影响分析**：由于偏移范围仅 3 cm，物体从最高 3 cm 落下会被重力拉到桌面。stability check 在 `reset_objects()` 之后运行，会等待物体稳定，因此短暂漂浮不会进入 episode。但严格来说，物体 Z 不随桌子移动可能导致边缘情况下的不一致。

## 6. 完整随机化时序

`BaseEnv.reset()` 中的随机化执行顺序（`base_env.py:672-715`）：

```
1. _randomize_table_height()          ← 桌面高度（最先，影响后续物理）
2. robot.reset()                       ← 可选 random_robot offset
3. reset_objects()                     ← 实例随机化 (model_id + scale)
   └─ _maybe_randomize_object() per object
4. scene_interact_with_objects()
5. stability check loop (retry)       ← 等物体稳定
6. _apply_domain_randomization()       ← 物理随机化 (friction + mass)
7. randomize_lighting()                ← 光照（若 enabled）
8. randomize_object_textures()         ← 物体纹理（若 enabled）
9. _capture_clutter_centers()
10. initialize_env_state()
11. contact_sensor.reset()
12. _randomize_skybox()                ← 天空盒（若 enabled）
13. _randomize_camera_pose()           ← 相机位姿（若 enabled）
14. _collect_scene_info()              ← 记录所有随机化结果
```

### 随机种子

所有随机化使用 `self.np_random`（numpy Generator），由 `reset(seed)` 统一播种。同一 seed 下所有随机化可复现。

### 物理随机化（独立于便捷开关）

`_apply_domain_randomization()` (`base_env.py:436-461`) 由 `PhysxConfig` 控制：

- `friction_range`: 随机静/动摩擦系数（static + dynamic，保证 dynamic ≤ static）
- `mass_scale_range`: 随机物体质量缩放因子

这些**不**受 `texture_random` 或 `instance_random` 控制，需独立配置。

## 7. 如何启用各类随机化

### CLI override

```bash
# table_random 和 instance_random 默认已开启（true），无需手动指定
# 仅需开启纹理随机化
bash scripts/eval_sim.sh dp3 pick_apple_messy <exp_dir> \
  'env_runner.env_kwargs.texture_random=true'

# 关闭默认开启的随机化
bash scripts/eval_sim.sh dp3 pick_apple_messy <exp_dir> \
  'env_runner.env_kwargs.table_random=false' \
  'env_runner.env_kwargs.instance_random=false'

# 全部随机化（三种都开启）
bash scripts/eval_sim.sh dp3 pick_apple_messy <exp_dir> \
  'env_runner.env_kwargs.texture_random=true'

# 使用便捷开关但覆盖某个子项
bash scripts/eval_sim.sh dp3 pick_apple_messy <exp_dir> \
  'env_runner.env_kwargs.texture_random=true' \
  'env_runner.env_kwargs.randomize_camera_pose=false'  # 关闭相机扰动
```

### 多任务

```bash
# 多任务: 所有任务统一
bash scripts/eval_sim.sh multitask_dit pick_bottle <exp_dir> \
  'env_runner.env_kwargs.instance_random=true'

# 多任务: 仅 pick_bottle 开启 (按 task_name 匹配)
# 需要在 task_configs 中按 index 或 task_name 指定
```
