# 配置系统

dexmani_policy 使用 Hydra + OmegaConf 作为配置框架，支持层级化 YAML 定义、`${...}` 变量插值、`${eval:'...'}` 数学表达式解析、CLI override 以及预设组合。

> 前置阅读：[01-项目概览](01-项目概览.md)
> 相关文档：[05-训练机制](05-训练机制.md)
> 快速导航：第 1 节 概览 · 第 4 节 插值系统 · 第 5 节 校验 · 第 6 节 策略对比 · 第 7 节 CLI override

## 1. 配置系统概览

```
configs/
├── dp.yaml                 # DP 策略（RGB + ConditionalUnet1D）
├── dp3.yaml                # DP3 策略（PC + ConditionalUnet1D）
├── maniflow.yaml           # ManiFlow 策略（PC + DiTX + FlowMatch）
├── maniflow_ddp.yaml       # └─ DDP 覆写（defaults: [maniflow]）
├── moe_dp3.yaml            # MoE 策略（PC + MoE encoder + ConditionalUnet1D）
├── multitask_dit.yaml      # MultiTask 策略（RGB + Text + DiT）
└── dataset/
    └── multitask_rgb.yaml  # 多任务数据集预设
```

### 1.1 配置层级

```
dataset/multitask_rgb.yaml  （数据集预设，可选）
        │
        ▼
dp.yaml / dp3.yaml / ...   （策略主配置，defaults 引入预设）
        │
        ▼
CLI overrides               （用户运行时覆写，最高优先级）
```

多任务配置使用了 Hydra 的 `defaults` 组合机制：

```yaml
# multitask_dit.yaml
defaults:
  - dataset: multitask_rgb    # 引入 dataset/multitask_rgb.yaml
  - _self_                    # 当前文件中的值覆盖 preset
```

DDP 配置则通过继承策略配置实现：

```yaml
# maniflow_ddp.yaml
defaults:
  - maniflow                  # 继承 maniflow.yaml 的所有配置
  - _self_                    # 当前文件的 training.num_gpus 等覆写
```

## 2. Hydra 入口机制

### 2.1 入口点

| 入口文件 | Hydra config_path | 说明 |
|---|---|---|
| `train.py` | `configs` | 单卡训练 |
| `train_ddp.py` | `configs` | 多卡 DDP 训练 |
| `train_multi_task.py` | `configs` | 多任务训练 |
| `eval_sim.py` | 读取实验目录 `config.yaml` | 独立评测（不使用 Hydra main） |

训练入口通过 `@hydra.main` 装饰器绑定：

```python
@hydra.main(version_base=None, config_path="configs")
def main(cfg):
    ...
```

Hydra 自动完成：
- 解析 `--config-name=<name>` → 加载 `configs/<name>.yaml`
- 处理 `defaults` 列表 → 合并 preset 配置
- 解析 `${...}` 变量插值
- 创建输出目录 `${hydra:runtime.output_dir}` → `experiments/<policy>/<task>/<timestamp>_<seed>/`
- 保存最终配置快照 `.hydra/config.yaml`（workspace 中另存一份 `config.yaml`）

### 2.2 输出目录规则

```yaml
hydra:
  run:
    dir: experiments/${policy_name}/${task_name}/${now:%Y-%m-%d_%H-%M}_${training.seed}
```

每次运行自动生成时间戳目录，例如：

```
experiments/dp3/pick_apple_messy/2026-04-01_11-18_42/
```

DDP 模式下，Hydra config 在主进程解析后通过 `OmegaConf.resolve(cfg)` 完全展开，再传入 `mp.spawn` 子进程（子进程无法访问 Hydra runtime resolver）。

## 3. 配置段结构

所有策略配置文件共享相同的顶层结构：

```yaml
policy_name: dp3              # 策略标识
task_name: pick_apple_messy   # 任务名
zarr_path: robot_data/sim/${task_name}.zarr

horizon: 16                   # 预测/观察窗口
n_obs_steps: 2
n_action_steps: 8
action_dim: 19

dataloader: { ... }           # DataLoader 参数
val_dataloader: { ... }       # 验证 DataLoader

dataset:                      # Hydra instantiate → Dataset
  _target_: ...
  ...

agent:                        # Hydra instantiate → Agent
  _target_: ...
  ...

optimizer: { ... }            # AdamW 参数
ema: { ... }                  # EMA 参数

workspace:                    # Hydra instantiate → TrainWorkspace
  _target_: ...
  wandb_cfg: { ... }
  checkpoint_cfg: { ... }

training:                     # 训练超参（非 instantiate）
  seed: ...
  device: cuda:0
  use_ema: true
  ...
  loop:
    num_epochs: 1000
    ...

env_runner:                   # Hydra instantiate → BaseRunner
  _target_: ...
  ...

eval:                         # 独立评测参数（非 instantiate）
  seed: 0                     # 固定值，不随 training.seed 变化
  offline:
    ckpt_tag_or_path: best
    ...

hydra: { ... }                # Hydra runtime 配置
```

### 3.1 `_target_` 实例化机制

带 `_target_` 的配置段通过 `hydra.utils.instantiate()` 实例化：

```python
# config.yaml
dataset:
  _target_: dexmani_policy.datasets.pc_dataset.PCDataset
  zarr_path: robot_data/sim/pick_apple_messy.zarr
  seed: 42
  ...

# train.py
dataset = hydra.utils.instantiate(cfg.dataset)
# → PCDataset(zarr_path="robot_data/...", seed=42, ...)
```

不带 `_target_` 的段（如 `training`、`eval`、`dataloader`）作为普通 dict 传递。

## 4. 变量插值系统

### 4.1 基本插值 `${...}`

配置内引用其他字段的值：

```yaml
seed: 42
zarr_path: robot_data/sim/${task_name}.zarr        # → robot_data/sim/pick_apple_messy.zarr
training:
  seed: ${seed}                                      # → 42
```

### 4.2 数学插值 `${eval:'...'}`

通过 `register_resolvers()`（`common/resolver.py`）注册，使用受限命名空间的安全 `eval`，支持数学表达式：

```yaml
pad_before: ${eval:'${n_obs_steps} - 1'}           # → 1
pad_after: ${eval:'${n_action_steps} - 1'}         # → 7
moe_out_dim: ${eval:'${agent.pc_out_dim} + ${agent.state_out_dim}'}  # → 192
```

语法规则：
- 表达式必须是 Python 表达式
- 内部 `${...}` 先被 OmegaConf 解析为数值，再传入 eval
- 支持算术（+-*/）、括号、比较、三元

### 4.3 Hydra 内置 Resolver

```yaml
output_dir: ${hydra:runtime.output_dir}             # Hydra 运行时输出目录
name: ${task_name}_${now:%Y-%m-%d_%H-%M}_${seed}   # 时间戳格式化
```

`${hydra:runtime.output_dir}` 由 Hydra 在运行时注入，指向当前实验输出路径。

## 5. 配置校验

`train.py::validate_config()` 在训练开始前执行，校验规则分四类：

### 5.1 窗口约束

```python
n_obs_steps <= horizon                    # 观察窗口不能超过预测窗口
n_action_steps <= horizon                 # 执行步数不能超过预测窗口
n_obs_steps - 1 + n_action_steps <= horizon  # 控制切片不能越界
```

### 5.2 优化器约束

```python
optimizer.obs_lr >= 0                     # 0 表示冻结 encoder（feature extractor 场景）
```

### 5.3 策略特定约束

| 策略 | 约束 |
|---|---|
| ManiFlow | `0 < flow_batch_ratio < 1`；`target_t_sample_mode ∈ {relative, absolute}`；`batch_size >= 2`（每个 micro-batch 独立做 flow/consistency split，不受 grad_accum_steps 影响）；`use_ema=false` 或 `use_ema_teacher_for_consistency=false` 时发出 UserWarning |
| MoE | `top_k <= num_experts` |
| MultiTask | `len(task_texts) == len(task_names)`；`len(task_weights) == len(task_names)`（如配置）；`len(env_runner.task_configs) == len(task_names)`；env_runner task_texts 必须在 dataset.task_texts 中；`persistent_workers=false`（epoch-dependent sampling 不兼容 persistent workers） |

### 5.4 跨模块一致性

```python
# PC 颜色增强需要 pc_dim >= 6（含 RGB 通道），否则 encoder 只读前 3 维
_validate_augmentation_consistency(cfg)

# modality_dropout 仅对归一化模态生效（joint_state, action）
# 非归一化模态（point_cloud, rgb）的配置会被忽略并发出警告

# MultiTask 数据集的 sampling_strategy 依赖 epoch-dependent 行为，
# 与 persistent_workers=True 不兼容
```

## 6. 策略配置对比

### 6.1 核心差异一览

| 配置键 | dp | dp3 | maniflow | moe_dp3 | multitask_dit |
|---|---|---|---|---|---|
| Dataset | `RGBDataset` | `PCDataset` | `PCDataset` | `PCDataset` | `MultiTaskDataset` |
| Sensor | rgb + joint | pc + joint | pc + joint | pc + joint | rgb + joint + text |
| Agent | `DPAgent` | `DP3Agent` | `ManiFlowAgent` | `MoEAgent` | `MultiTaskAgent` |
| Encoder | DINO/CLIP | iDP3/PointNeXT | PointPN | iDP3 + MoE | DINO + CLIP Text |
| Backbone | `ConditionalUnet1D` | `ConditionalUnet1D` | `DiTXFlowMatch` | `ConditionalUnet1D` | `DiT_Diffusion` |
| Decoder | `Diffusion` | `Diffusion` | `FlowMatchWithConsistency` | `Diffusion` | `Diffusion` |
| EMA teacher | - | - | ✓ | - | - |

### 6.2 训练超参差异

| 配置键 | dp | dp3 | maniflow | moe_dp3 | multitask_dit |
|---|---|---|---|---|---|
| batch_size | 256 | 256 | 128 | 256 | 64 |
| lr | 1e-4 | 1e-4 | 1e-4 | 1e-4 | 1e-4 |
| weight_decay | 1e-6 | 1e-6 | **1e-3** | 1e-6 | 1e-6 |
| betas | (0.95, 0.999) | (0.95, 0.999) | **(0.9, 0.95)** | (0.95, 0.999) | (0.95, 0.999) |
| eval_interval_epochs | 250 | 250 | **200** | 250 | 250 |
| use_ema_teacher_for_consistency | false | false | **true** | false | false |
| modality_dropout | - | - | state: 0.2 | - | - |

### 6.3 DDP 配置 (maniflow_ddp.yaml)

```yaml
defaults:
  - maniflow
  - _self_

policy_name: maniflow_ddp

training:
  num_gpus: 4
  gpu_ids: null

dataloader:
  batch_size: 32      # 每卡 batch，总 batch = 32 × 4 = 128
  num_workers: 4      # 每卡 worker 数
```

关键点：
- `defaults: [maniflow]` 继承 `maniflow.yaml` 全部配置
- `training.num_gpus` 和 `training.gpu_ids` 是 DDP 专属字段（`train.py` 不读取）
- 总 batch_size = `dataloader.batch_size × num_gpus`

## 7. CLI Override 语法

Hydra 支持从命令行覆写任意配置字段：

```bash
# 修改训练 epoch 数
bash scripts/train.sh dp3 'training.loop.num_epochs=100'

# 修改学习率
bash scripts/train.sh dp3 'optimizer.lr=3e-5'

# 修改 batch size 和 worker 数
bash scripts/train.sh dp3 'dataloader.batch_size=128' 'dataloader.num_workers=4'

# 修改任务名和 zarr 路径（同时影响所有 ${task_name} 引用）
bash scripts/train.sh dp3 'task_name=pour'

# 评测时修改 checkpoint 和评测参数
bash scripts/eval_sim.sh maniflow pick_apple_messy 2026-04-01_11-18 \
    'eval.offline.ckpt_tag_or_path=epoch=0100-step=0012345-score=0.7500.pt' \
    'eval.offline.denoise_timesteps_list=[5,10,20]'

# 嵌套 override
bash scripts/train.sh dp3 'agent.rgb_backbone_name=clip' 'agent.rgb_backbone_config.tune_mode=full'

# 修改 wandb 模式
bash scripts/train.sh dp3 'workspace.wandb_cfg.mode=online'
```

## 8. 配置解析时序

```
1. Hydra 加载 configs/<name>.yaml
2. 处理 defaults 列表，合并 preset
3. ${...} 插值解析（包括跨文件引用）
4. ${eval:'...'} 数学插值解析
5. ${hydra:...} Hydra runtime 解析
6. CLI override 覆写
7. OmegaConf.to_container(resolve=True) 完全展开（DDP 模式）
8. hydra.utils.instantiate() 实例化 _target_ 段
9. validate_config() 校验 → 通过后开始训练
```

DDP 模式在第 7 步后通过 `mp.spawn` 传递已解析的 cfg 给子进程，子进程不再依赖 Hydra resolver。

## 9. 新增策略配置 checklist

1. 在 `configs/` 创建 `<name>.yaml`
2. 设置 `policy_name`、`_target_`（Dataset / Agent / env_runner）
3. 配置 `sensor_modalities` 与 agent 感知模态一致
4. 如需要 DDP，创建 `<name>_ddp.yaml` 通过 `defaults: [<name>]` 继承
5. 在 `train.py::validate_config()` 中按需添加策略特定校验
6. 在 `scripts/train.sh` 的 usage 中补充说明
