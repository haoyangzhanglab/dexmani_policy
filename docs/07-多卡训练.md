# 多卡训练

dexmani_policy 通过 PyTorch DistributedDataParallel (DDP) + `torch.multiprocessing.spawn` 实现多卡训练。DDP 训练复用单卡 `Trainer`，通过 `DDPTrainer` 包装器处理分布式逻辑。

## 1. 整体架构

```
train_ddp.py (main process)
  │
  ├─ validate_config(cfg)
  ├─ OmegaConf.resolve(cfg)          # 必须在 spawn 前完全展开所有插值
  │
  └─ mp.spawn(ddp_worker, nprocs=N)  # 启动 N 个子进程
       │
       ├─ [Rank 0]                     ├─ [Rank 1]                    ├─ [Rank N-1]
       │  setup_ddp()                  │  setup_ddp()                 │  setup_ddp()
       │  dataset (seed+0)             │  dataset (seed+1)            │  dataset (seed+N-1)
       │  DistributedSampler           │  DistributedSampler          │  DistributedSampler
       │  model + EMA + opt            │  model + EMA + opt           │  model + EMA + opt
       │  DDP(model, device_ids=[0])   │  DDP(model, device_ids=[1])  │  DDP(model, device_ids=[N-1])
       │  TrainWorkspace (完整)        │  workspace=None              │  workspace=None
       │                               │  CheckpointStore (只读)     │  CheckpointStore (只读)
       │  env_runner                   │  env_runner=None             │  env_runner=None
       │  val_loader                   │  val_loader=None             │  val_loader=None
       │                               │                              │
       │  DDPTrainer.train()           │  DDPTrainer.train()          │  DDPTrainer.train()
       │   ├─ 数据并行训练             │   ├─ 数据并行训练             │   ├─ 数据并行训练
       │   ├─ logging                  │   ├─ logging (skip)          │   ├─ logging (skip)
       │   ├─ eval                     │   ├─ eval (skip)             │   ├─ eval (skip)
       │   └─ checkpoint               │   └─ checkpoint (skip)       │   └─ checkpoint (skip)
       │  cleanup_ddp()                │  cleanup_ddp()               │  cleanup_ddp()
```

## 2. 进程启动与初始化

### 2.1 入口点 `train_ddp.py`

```python
# 必须使用 spawn（CUDA fork 不安全）
mp.set_start_method("spawn", force=True)

@hydra.main(version_base=None, config_path="configs")
def main(cfg):
    validate_config(cfg)

    # CUDA 可用性检查
    if not torch.cuda.is_available():
        raise RuntimeError("CUDA is not available. DDP training requires GPU.")

    num_gpus = cfg.training.get("num_gpus")
    if num_gpus is None:
        raise ValueError(
            "train_ddp.py requires 'training.num_gpus' to be set in config."
        )
    if num_gpus <= 1:
        raise ValueError(
            f"train_ddp.py requires num_gpus > 1, got {num_gpus}. "
            f"Please use train.py for single-GPU training."
        )

    # 校验 gpu_ids 与可用 GPU 数量
    available_gpus = torch.cuda.device_count()
    gpu_ids = cfg.training.get("gpu_ids", None)
    if gpu_ids is not None:
        if len(gpu_ids) != num_gpus:
            raise ValueError(...)
        for gpu_id in gpu_ids:
            if gpu_id >= available_gpus:
                raise ValueError(...)
    else:
        if num_gpus > available_gpus:
            raise ValueError(...)
        gpu_ids = list(range(num_gpus))

    # 设置 MASTER_ADDR / MASTER_PORT（见 2.3 节）

    # 关键：子进程无法访问 Hydra runtime resolver，必须在 spawn 前解析所有插值
    OmegaConf.resolve(cfg)

    mp.spawn(
        ddp_worker,
        args=(num_gpus, cfg, gpu_ids),    # 传递已解析的 cfg
        nprocs=num_gpus,
        join=True
    )
```

### 2.2 GPU 分配

支持通过 `gpu_ids` 指定物理 GPU，或自动使用前 N 张卡：

```yaml
# maniflow_ddp.yaml
training:
  num_gpus: 4
  gpu_ids: null             # 指定使用哪几张GPU，null表示使用前num_gpus张卡
```

在 `main()` 中对 `gpu_ids` 进行严格校验（长度匹配、GPU 可用性），若未指定则 fallback 为 `list(range(num_gpus))`。校验后的 `gpu_ids` 通过 `args` 传入 `ddp_worker`：

```python
actual_gpu_id = gpu_ids[rank] if gpu_ids else rank
device = torch.device(f'cuda:{actual_gpu_id}')
torch.cuda.set_device(device)
```

### 2.3 MASTER_ADDR / MASTER_PORT

若未设置，自动配置 localhost + 随机端口：

```python
if 'MASTER_ADDR' not in os.environ:
    os.environ['MASTER_ADDR'] = 'localhost'
if 'MASTER_PORT' not in os.environ:
    # 分配随机可用端口，避免多组实验冲突
    sock = socket.socket()
    sock.bind(('', 0))
    os.environ['MASTER_PORT'] = str(sock.getsockname()[1])
    sock.close()
```

### 2.4 进程组

```python
def setup_ddp(rank, world_size):
    dist.init_process_group(
        backend='nccl',
        init_method='env://',
        world_size=world_size,
        rank=rank
    )
```

## 3. 数据分片

### 3.1 DistributedSampler

DDP 下采用两阶段 seed 设置：模型初始化前所有 rank 使用相同 seed 保证初始权重一致，模型构建后各 rank 使用不同 seed 增加数据增强多样性：

```python
# 阶段 1: 相同 seed — 确保所有 rank 模型初始参数一致（DDP 要求）
set_seed(cfg.training.seed)

# 构建 dataset 和 normalizer（DDP 模式下 resolve 配置）
dataset, normalizer = build_dataset_and_normalizer(cfg, rank=rank, world_size=world_size)

# DistributedSampler 负责分片（在模型构建前创建，此时 seed 仍相同）
train_sampler = DistributedSampler(
    dataset,
    num_replicas=world_size,
    rank=rank,
    shuffle=cfg.dataloader.shuffle,
    seed=cfg.training.seed,
)

train_loader = DataLoader(
    dataset,
    sampler=train_sampler,    # sampler 和 shuffle 互斥
    batch_size=...,           # 每卡 batch size
    ...
)

# 构建 model 和 EMA（此时所有 rank 权重完全相同）
model, ema_model, ema_updater = build_model_and_ema(cfg, device, normalizer)

# 阶段 2: 不同 seed — 各 rank 数据增强多样性（DataLoader workers 通过 worker_init_fn 获取 seed）
set_seed(cfg.training.seed + rank)

# 构建 optimizer 和 scheduler
batches_per_epoch = len(train_loader)
optimizer, scheduler = build_optimizer_and_scheduler(cfg, model, batches_per_epoch)
```

**关键点**：`dataloader.batch_size` 是每张卡的 batch size，总 batch = `batch_size × num_gpus`。梯度累积在此基础上进一步放大有效 batch。

### 3.2 验证集与评测隔离

验证集和评测只在 rank 0 创建，env_runner 需要同时满足 `eval_interval_epochs > 0` 且 `cfg.env_runner` 存在：

```python
# rank 0 独享
val_loader = DataLoader(val_dataset, ...)    # rank 0 only
env_runner = None
if rank == 0 and cfg.training.loop.eval_interval_epochs > 0 and cfg.get("env_runner") is not None:
    env_runner = hydra.utils.instantiate(cfg.env_runner)    # rank 0 only
workspace = TrainWorkspace(...)              # rank 0: 完整版
checkpoint_store = workspace.checkpoint_store

# rank > 0
val_loader = None
env_runner = None
workspace = None                             # rank > 0: 不创建 workspace
checkpoint_store = CheckpointStore(checkpoint_dir)  # 只读 checkpoint
```

非 rank-0 进程只需读取 checkpoint 恢复训练状态，通过 `CheckpointStore` 直接完成，无需 workspace 桩对象。

## 4. 模型包装与同步

### 4.1 DDP 包装

只包装训练模型，EMA 模型不参与 DDP：

```python
class DDPTrainer:
    def __init__(self, ...):
        self.raw_model = model                    # 保存未包装的引用（用于 checkpoint）
        ddp_model = DDP(model,
            device_ids=[actual_gpu_id],
            output_device=actual_gpu_id,
            find_unused_parameters=False          # 关闭未使用参数检测以提升性能
        )
        self.trainer = Trainer(
            model=ddp_model,                      # 传入 DDP 包装后的模型
            ema_model=ema_model,                  # EMA 不包装
            ...
        )
```

- `self.raw_model` → 未包装模型，checkpoint 保存其 state_dict
- `ddp_model` → DDP 包装，用于 `forward` 和 `backward`
- `find_unused_parameters=False`：模型所有参数均参与计算，关闭检测可减少 DDP 开销
- EMA 模型不包装：不参与梯度计算，不需要 DDP 同步

### 4.2 Normalizer 同步

各 rank 从各自数据分片独立拟合 normalizer，rank 0 的结果通过 `broadcast` 同步给其他 rank：

```python
def synchronize_states(self):
    """rank 0 broadcast normalizer state_dict 到所有进程"""
    norm_state = self.raw_model.normalizer.state_dict()
    for key in norm_state:
        if isinstance(norm_state[key], torch.Tensor):
            dist.broadcast(norm_state[key], src=0)
    if self.rank != 0:
        self.raw_model.normalizer.load_state_dict(norm_state)
```

Optimizer 和 Scheduler 不需要同步：各 rank 独立初始化，DDP 梯度同步保证参数更新一致。

### 4.3 梯度同步

DDP 自动在 `loss.backward()` 时通过 all-reduce 同步梯度。梯度累积的处理：

```python
# loss 除以 grad_accum_steps（DDP 梯度是 all-reduce 的，不需手动 /world_size）
scaled_loss = raw_loss / grad_accum_steps
scaled_loss.backward()

accum_step += 1
if accum_step % grad_accum_steps == 0:
    apply_gradient_step()   # clip_grad_norm → optimizer.step → scheduler.step → EMA
```

### 4.4 Epoch 边界同步

每个 epoch 开始时重置梯度累积状态，epoch 结束通过 `dist.barrier()` 同步所有 rank。rank 0 的 finish_epoch 异常通过 `error_flag` tensor 广播给所有 rank：

```python
for epoch in epoch_pbar:
    train_sampler.set_epoch(epoch)    # 确保每 epoch 数据 shuffle 不同
    trainer.on_epoch_start(epoch)     # 调用 dataset/model 的 set_epoch hook

    trainer.model.train()

    # 避免跨 epoch 边界的梯度累积错位
    trainer.accum_step = 0
    trainer.optimizer.zero_grad(set_to_none=True)

    # ... training loop (train_one_step × N) ...

    # epoch 结束时 flush 未完成的梯度累积，避免浪费计算
    trainer.flush_gradient_accumulation()

    trainer.model.eval()

    # rank 0 独享 logging/checkpoint/eval
    finish_error = None
    if is_main:
        try:
            trainer.finish_epoch(
                epoch, global_step,
                last_batch=last_batch,
                eval_model=(ema_model if use_ema else raw_model),
                checkpoint_model=raw_model,
            )
        except Exception as e:
            import traceback
            traceback.print_exc()
            print("[Rank 0] finish_epoch failed, terminating.")
            finish_error = e

    # 将 rank 0 的错误状态广播给所有 rank
    error_flag = torch.tensor(
        [1 if finish_error is not None else 0],
        dtype=torch.int, device=trainer.device,
    )
    dist.broadcast(error_flag, src=0)
    dist.barrier()

    if error_flag.item():
        raise RuntimeError(
            "Training aborted due to error in finish_epoch on rank 0"
        ) from (finish_error if finish_error is not None else None)
```

`train_sampler.set_epoch(epoch)` 是关键调用：它改变 `DistributedSampler` 内部的 shuffle 种子，确保每个 epoch 的数据划分不同。

## 6. Checkpoint 兼容性

### 6.1 保存格式

Checkpoint 始终以 `self.raw_model`（未包装）的 state_dict 保存：

```python
# finish_epoch → TrainCheckpoint(model_state=self.raw_model.state_dict())
checkpoint = TrainCheckpoint(
    model_state=checkpoint_model.state_dict(),   # raw_model
    ema_model_state=self.ema_model.state_dict(),
    ...
)
```

### 6.2 `fix_state_dict()` 跨模式转换

```python
# 单卡 checkpoint → DDP 模型加载
model_state = fix_state_dict(checkpoint.model_state, is_current_ddp=True)
# → 自动添加 "module." 前缀

# DDP checkpoint → 单卡模型加载
model_state = fix_state_dict(checkpoint.model_state, is_current_ddp=False)
# → 自动移除 "module." 前缀

# EMA 模型始终非 DDP
ema_state = fix_state_dict(checkpoint.ema_model_state, is_current_ddp=False)
```

关键实现位于 `dexmani_policy/common/pytorch_util.py`：

```python
def fix_state_dict(state_dict: Dict, is_current_ddp: bool) -> Dict:
    # 仅检查第一个 key 是否带 module. 前缀即可判断 checkpoint 格式
    first_key = next(iter(state_dict.keys()))
    is_checkpoint_ddp = first_key.startswith('module.')

    if is_checkpoint_ddp and not is_current_ddp:
        return {k.removeprefix("module."): v for k, v in state_dict.items()}

    elif not is_checkpoint_ddp and is_current_ddp:
        return {f'module.{k}': v for k, v in state_dict.items()}

    return state_dict
```

### 6.3 恢复训练

DDP 恢复时先以非 DDP 格式加载到 `raw_model`（DDP 已在 `__init__` 中完成包装）。加载后同步 normalizer 到所有 rank，并 `dist.barrier()` 确保所有 rank 就绪：

```python
def train(self, resume_tag: str = "latest"):
    torch.set_float32_matmul_precision('high')
    try:
        checkpoint = self.checkpoint_store.load(
            self.checkpoint_store.resolve_path(resume_tag)
        )

        # 先加载到 raw_model（非 DDP 格式）
        model_state = fix_state_dict(checkpoint.model_state, is_current_ddp=False)
        self.raw_model.load_state_dict(model_state, strict=True)

        # 加载 EMA（始终非 DDP）
        if self.trainer.use_ema and checkpoint.ema_model_state is not None:
            ema_state = fix_state_dict(checkpoint.ema_model_state, is_current_ddp=False)
            self.trainer.ema_model.load_state_dict(ema_state, strict=True)

        # 加载 optimizer / scheduler / ema_updater
        self.trainer.optimizer.load_state_dict(checkpoint.optimizer_state)
        self.trainer.scheduler.load_state_dict(checkpoint.scheduler_state)
        if self.trainer.ema_updater is not None and checkpoint.ema_updater_state is not None:
            self.trainer.ema_updater.load_state_dict(checkpoint.ema_updater_state)

        global_step = checkpoint.global_step
        start_epoch = checkpoint.epoch + 1
    except FileNotFoundError:
        global_step = 0
        start_epoch = 0

    # 将 optimizer 状态移到正确设备
    optimizer_to(self.trainer.optimizer, self.trainer.device)
    if self.trainer.use_ema and self.trainer.ema_model is not None:
        self.trainer.ema_model.to(self.trainer.device)

    # 同步 normalizer 到所有 rank，broadcast 后 barrier 保证所有 rank 就绪
    self.synchronize_states()
    dist.barrier()
```

## 7. 梯度累积与 DDP 交互

梯度累积在 DDP 下正常工作，因为：
- 每次 `loss.backward()` 累加梯度（不清零）
- DDP 的 all-reduce 在 `backward()` 时自动触发
- 梯度缩放 `p.grad.mul_(scale)` 在 flush 时应用

```python
def flush_gradient_accumulation(self):
    if accum_step % grad_accum_steps == 0:
        return  # 刚好被整除，无需 flush

    # 最后一批样本少于完整累积步数，缩放梯度以保持更新等价
    scale = grad_accum_steps / (accum_step % grad_accum_steps)
    for p in self.model.parameters():
        if p.grad is not None:
            p.grad.mul_(scale)

    self.apply_gradient_step()
```

## 8. 性能考量

### 8.1 吞吐量估算

| 配置 | 单卡 batch_size | GPU 数 | 总 batch | 有效 batch (grad_accum=1) |
|---|---|---|---|---|
| maniflow (单卡) | 128 | 1 | 128 | 128 |
| maniflow_ddp | 32 | 4 | 128 | 128 |

DDP 配置下，每卡 batch_size=32 使 total batch=128（与单卡等价），梯度累积步数不变时每步计算量相同但 I/O 更均衡。

### 8.2 开销来源

| 开销 | 说明 |
|---|---|
| all-reduce 梯度同步 | 每个 backward 后触发，对 768 维 hidden 的模型而言开销较小 |
| `dist.barrier()` | 每 epoch 一次，等待 rank 0 完成 checkpoint/val/eval |
| 数据加载 | 每卡独立 workers，总 I/O = workers × num_gpus |
| NCCL 初始化 | 仅启动时一次 |

### 8.3 多任务 DDP

当前 DDP 训练仅支持单任务场景。多任务训练 (`train_multi_task.py`) 使用 `MultiTaskDataset`，其内部的 `MultiTaskDataset.get_validation_dataset()` 返回另一个 `MultiTaskDataset`，未集成 DDP 支持。

## 9. 常见问题

**Q: 为什么 DDP 必须用 spawn 而不是 fork？**

CUDA 运行时在 fork 后状态未定义，可能导致死锁或 crash。`mp.set_start_method("spawn", force=True)` 确保子进程重新初始化 CUDA 上下文。

**Q: 为什么 `OmegaConf.resolve(cfg)` 必须在 spawn 前调用？**

子进程无法访问 Hydra 的 runtime resolver（`${hydra:...}`、`${now:...}` 等），必须在父进程中完全展开。

**Q: 为什么 normalizer 需要 broadcast 同步？**

每个 rank 从不同的分片数据拟合 normalizer，如果不同步会导致各 rank 的归一化参数不一致，梯度更新会发散。

**Q: 从单卡切换 DDP 需要改哪些配置？**

1. 使用 `train_ddp.py` 代替 `train.py`
2. 使用 DDP config（如 `maniflow_ddp.yaml`）或添加 `training.num_gpus`
3. 确认 `dataloader.batch_size` 是每卡的值（不是总 batch）

**Q: DDP checkpoint 能给单卡推理用吗？**

能。checkpoint 始终以非 DDP 格式保存，`fix_state_dict(is_current_ddp=False)` 保证正确加载。

**Q: MoE + DDP 训练有什么注意事项？**

`DDPTrainer` 默认 `find_unused_parameters=False` 以提升性能。若 MoE 使用 `top_k < num_experts` 的稀疏专家激活，部分 expert 参数在某些 rank 上可能未被使用，DDP 会报错。当前无 `moe_dp3_ddp` 配置，若将来添加需将 `find_unused_parameters` 设为 True 或确保 `top_k = num_experts`。
