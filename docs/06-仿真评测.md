两条评测路径的对比：

| 维度 | 训练期评测 (Trainer.evaluate) | 独立评测 (eval_sim.py) |
|---|---|---|
| 入口 | `trainer.py:186` `evaluate()` | `eval_sim.py:21` `run_eval()` |
| episode 数 | `env_runner.default_eval_episodes` (25/20) | `cfg.eval.offline.eval_episodes` (100) |
| denoise 步数 | action_decoder 默认 (Diffusion: `num_inference_steps=10`, FlowMatch: `denoise_timesteps=10`) | `cfg.eval.offline.denoise_timesteps_list` ([10]) |
| 模型权重 | EMA (`self.ema_model`) | EMA (`use_ema_for_eval: true`) |
| checkpoint | — | `ckpt_tag_or_path: best` (topk manifest) |
| 输出 | wandb metrics / jsonl | video mp4 + metrics.json 到 `eval/<timestamp>/` |
| 调度 | `eval_interval_epochs` 控制 | 手动 CLI 触发 |

## 2. 评测工作流

### 2.1 完整调用链

```
eval_sim.sh dp3 pick_apple_messy <exp_dir>
  │
  ├─ 1. 加载配置
  │     exp_dir/config.yaml → OmegaConf.load()
  │     合并 CLI overrides (OmegaConf.merge)
  │     校验 cfg.eval.offline 存在
  │
  ├─ 2. 构建组件
  │     set_seed(cfg.eval.seed)
  │     agent = hydra instantiate cfg.agent
  │     env_runner = hydra instantiate cfg.env_runner
  │     checkpoint_store = CheckpointStore(exp_dir / "checkpoints")
  │
  ├─ 3. SimEvaluator.run()
  │     │
  │     ├─ 3a. 加载 checkpoint
  │     │     resolve_path("best")
  │     │       → topk_manifest.json → items[0]["path"]
  │     │     CheckpointStore.load(path) → torch.load(weights_only=False)
  │     │     _load_for_inference(ckpt_tag_or_path, use_ema_for_eval)
  │     │       内含: train_params 校验 + fix_state_dict + normalizer 校验
  │     │
  │     ├─ 3b. 创建输出目录
  │     │     <exp_dir>/eval/<timestamp>/
  │     │     SimEvalRecorder 保存 eval_config.json
  │     │
  │     └─ 3c. denoise_timesteps 扫描
  │           for dt in denoise_timesteps_list:
  │             env_runner.run(agent, dt, eval_episodes)
  │               │
  │               ├─ make_env()
  │               │   importlib.import_module("dexmani_sim.envs.pick_apple_messy")
  │               │   getattr(module, "PickAppleMessy")
  │               │   env = PickAppleMessy(render_mode="rgb_array")
  │               │
  │               ├─ get_seed_list()
  │               │   DATA_DIR/eval_seeds/pick_apple_messy.txt
  │               │   → [2101, 4073, 2624, ...]   (120 个 seed)
  │               │
  │               └─ for seed in seeds[:eval_episodes]:
  │                     eval_one_episode(agent, env, seed, dt)
  │                       │
  │                       ├─ env.reset(seed=seed)
  │                       │   └─ robot.reset() → 重置关节
  │                       │      reset_objects() → 随机采样物体位置
  │                       │      check_stable() → 物理稳定性检测
  │                       │      不稳定 → RuntimeError("Reset Failed") → 跳过 seed
  │                       │
  │                       ├─ update_obs(obs)  → obs_deque.append(obs)
  │                       │
  │                       └─ while not truncated:
  │                            get_obs_batch()
  │                              stack_last_n(deque, n_obs_steps=2)
  │                              → {"point_cloud": (2,1024,D), ...}
  │                              unsqueeze(0) → batch dim
  │                            agent.predict_action(obs_batch, denoise_timesteps)
  │                              _build_cond()
  │                                normalize → truncate(2)→ flatten(0,1)
  │                                obs_encoder → cond
  │                              action_decoder.predict_action(cond, ...)
  │                                [Diffusion]  DDIM 去噪 (num_inference_steps 步)
  │                                [FlowMatch]  Euler ODE 积分 (denoise_timesteps 步)
  │                              normalizer['action'].unnormalize(pred)
  │                              → control_action: (8,19)
  │                            for i in range(8):
  │                              env.step(action_chunk[i])
  │                                frame_skip=20 次物理步进
  │                                apply_action(target_qpos) → PD 控制器
  │                                scene.step() → PhysX 仿真
  │                                get_sensor_data() + preprocess_obs_data()
  │                                compute_metric_and_condition() → reward, success
  │                                compute_done() → consistent_success_count >= 3
  │                                compute_truncated() → action_cnt >= limit
  │                            → (done, task_done_step)
  │
  │                     video = env.get_video()  → ndarray (N_frames, H, W, 3)
  │                     收集 success_list, episode_details
  │
  │             recorder.save_case_result(result, dt)
  │               → denoise_timesteps<N>/eval_episode_<seed>.mp4
  │               → denoise_timesteps<N>/metrics.json
  │
  └─ 4. recorder.save_summary(eval_metrics.json)
       → {meta, metrics: {"denoise_timesteps10": {...}}}
```

## 3. 评测入口

### 3.1 独立评测 CLI

`eval_sim.py` 负责从已完成的实验目录加载配置和 checkpoint，执行离线评测。

```bash
# 基本用法
bash scripts/eval_sim.sh <policy_name> <task_name> <exp_name> [overrides...]

# 示例
bash scripts/eval_sim.sh dp3 pick_apple_messy 2026-04-01_11-18_233
bash scripts/eval_sim.sh maniflow pour latest_exp \
  eval.offline.denoise_timesteps_list='[5,10,20]' \
  eval.offline.eval_episodes=200
```

CLI 参数：

| 参数 | 说明 | 示例 |
|---|---|---|
| `--policy-name` | 策略名，用于定位实验目录 | `dp3`, `maniflow`, `moe_dp3` |
| `--task-name` | 任务名，用于定位实验目录 | `pick_apple_messy`, `pour` |
| `--exp-name` | 实验名，实验目录的最后一层 | `2026-04-01_11-18_233` |
| `[overrides]` | Hydra 风格 override | `eval.offline.eval_episodes=200` |

实验目录解析：
```python
exp_dir = ROOT_DIR / "experiments" / policy_name / task_name / exp_name
# 例如: experiments/dp3/pick_apple_messy/2026-04-01_11-18_233
```

### 3.2 训练期评测

在 `Trainer.train()` 的训练循环中，按 `eval_interval_epochs` 周期触发评测。

```python
# trainer.py:75 plan_epoch_end_tasks()
"evaluate": self.enable_env_eval and (is_last or (epoch + 1) % self.eval_interval_epochs == 0)

# trainer.py:199 finish_epoch()
if eval_model is None:
    eval_model = self.ema_model if self.use_ema else self.model
# ...
if epoch_end_tasks["evaluate"]:
    eval_metrics = self.evaluate(eval_model)  # 使用 EMA 模型
    epoch_metrics.update(eval_metrics)
```

与独立评测的关键区别：
- **不传 `denoise_timesteps`**：使用 action_decoder 的默认推理步数
- **不传 `eval_episodes`**：使用 `env_runner.default_eval_episodes`（各策略 20 或 25）
- **结果写 wandb / jsonl**：不保存视频到磁盘，但会以 wandb.Video 方式记录
- **success_rate → checkpoints 排序**：`test_mean_score` 直接使用 `eval/success_rate`，TopK tracker 据此保留最优 checkpoint

## 4. SimEvaluator 与 SimEvalRecorder

`SimEvaluator` 是独立评测的编排器，负责 checkpoint 加载、denoise_timesteps 扫描和结果汇总。

```python
# training/sim_evaluator.py
class SimEvaluator:
    def __init__(self, device, agent, env_runner, checkpoint_store, eval_root_dir):
        self.eval_root_dir = eval_root_dir   # 由调用方传入，如 exp_dir / "eval"
        ...

    def run(self, eval_episodes, denoise_timesteps_list, ckpt_tag_or_path, use_ema_for_eval, eval_config):
        if not denoise_timesteps_list:
            raise ValueError("denoise_timesteps_list cannot be empty.")
        # 1. 加载 checkpoint
        self._load_for_inference(ckpt_tag_or_path, use_ema_for_eval)
        agent.to(device); agent.eval()

        # 2. 创建时间戳输出目录
        eval_run_dir = eval_root_dir / "<timestamp>"   # 例如 20260514_143022/

        # 3. 遍历 denoise_timesteps
        for denoise_timesteps in denoise_timesteps_list:
            result = env_runner.run(agent, denoise_timesteps, eval_episodes)
            recorder.save_case_result(result, denoise_timesteps)

        # 4. 汇总结果
        recorder.save_summary(summary)   # 保存为 eval_metrics.json
```

`SimEvalRecorder` 负责视频和指标的持久化：

### 5.1 数据结构

```python
class BaseRunner:
```

### 5.3 单 episode 循环

```python
def eval_one_episode(self, agent, env, episode_seed, denoise_timesteps=None, **kwargs):
    obs, info = env.reset(seed=episode_seed, options=kwargs.get("options", None))
    self.reset()               # 清空 obs_deque + 重置 dropped_keys_warned 标志
    self.update_obs(obs)       # deque = [obs_0]

    truncated = False
    done = False
    prev_done = False
    task_done_step = None

    while not truncated:
        obs = self.get_obs_batch(device)        # → (1, 2, *) batch
        action_chunk = self.get_action_chunk(obs, agent, denoise_timesteps)
        # action_chunk: np.ndarray (8, 19)

        for i in range(action_chunk.shape[0]):  # 8 步
            obs, reward, done, truncated, info = env.step(action_chunk[i])
            self.update_obs(obs)

            if done and not prev_done:
                task_done_step = getattr(env, 'action_cnt', None)  # 安全访问：action_cnt 为可选属性
            prev_done = done

            if truncated:
                break

    return done, task_done_step
```

### 5.4 episode 运行与异常处理

```python
def run(self, agent, denoise_timesteps, eval_episodes):
    env = self.make_env()
    eval_seeds = self.get_seed_list()
    num_episodes = eval_episodes or self.default_eval_episodes
    # 若 eval_episodes > 可用种子数则截断并警告

    try:
        while len(success_list) < num_episodes:
            eval_seed = eval_seeds[seed_idx]; seed_idx += 1
            try:
                done, task_done_step = self.eval_one_episode(...)
                video = env.get_video()
                completed += 1
                # 每 clear_cache_freq 个 episode 关闭并重建环境防止显存累积
                if completed % clear_cache_freq == 0:
                    env.close(); env = self.make_env()

                success_list.append(done)
                episode_details.append({"seed": seed, "success": done, "steps": task_done_step})
                if video is not None:
                    episode_video_list.append(...)
                # ...
            except (RuntimeError, ValueError, AttributeError) as e:
                # AttributeError: 动态导入的 env 可能缺失可选属性（如 action_cnt）
                if "Reset Failed" in str(e) or "Unstable" in str(e) or "Failed to sample" in str(e):
                    # 环境初始化失败 → 跳过该 seed，不计入失败
                    env_failed_seeds.append(eval_seed)
                else:
                    # 策略执行失败 → 记录为失败 episode
                    success_list.append(False)
            except KeyboardInterrupt:
                break  # 用户中断，返回部分结果
            except Exception:
                # 未预期的严重错误，打印详细信息后重新抛出
                import traceback
                traceback.print_exc()
                raise
    finally:
        env.close()

    success_rate = np.mean(success_list) if success_list else None
    avg_steps = int(round(np.mean(task_done_step_list))) if task_done_step_list else None
    return {"success_rate": success_rate, "avg_steps": avg_steps,
            "videos": episode_video_list, "episode_details": episode_details}
```

异常处理的三级分类：

| 异常类型 | 匹配条件 | 处理 | 对指标的影响 |
|---|---|---|---|
| 环境初始化失败 | `"Reset Failed"`, `"Unstable"`, `"Failed to sample"` in error_msg | 跳过 seed，不增加 episode 计数 | 无（seed 被丢弃） |
| 策略执行失败 | 其他 RuntimeError / ValueError | 记录为 `success=False` | 降低 success_rate |
| 用户中断 | KeyboardInterrupt | 立即停止循环，返回已有结果 | 只汇报已完成的 episode |
| 未预期错误 | 其他 Exception | 打印 traceback 后重新抛出 | 评测中断 |

## 6. SimRunner — 环境适配层

`SimRunner` 继承 `BaseRunner`，实现两个抽象方法 `make_env()` 和 `get_seed_list()`。

### 6.1 环境动态导入

```python
class SimRunner(BaseRunner):
    def make_env(self):
        # task_name: "pick_apple_messy" (来自 config)
        env_module = importlib.import_module(f"dexmani_sim.envs.{self.task_name}")
        env_class = getattr(env_module, self.name_to_pascal_case(self.task_name))
        # "pick_apple_messy" → "PickAppleMessy"
        return env_class(render_mode="rgb_array", **self.env_kwargs)

    @staticmethod
    def name_to_pascal_case(name: str) -> str:
        # "pick_apple_messy" → "PickAppleMessy"
        # "cube_in_box"      → "CubeInBox"
        return ''.join(part.capitalize() for part in re.split(r'[_\s-]+', name) if part)
```

评测时使用 `render_mode="rgb_array"`（非 `"human"`），无 viewer 窗口，视频帧来自 `obs["rgb"]`。

### 6.2 种子文件

```python
def get_seed_list(self) -> List[int]:
    # 优先级: 构造函数传入的 eval_seeds > seed 文件 > fallback range(100)
    seed_file = DATA_DIR / "eval_seeds" / f"{self.task_name}.txt"
    # 例如: ~/Desktop/dexmani_sim/mimic_data/eval_seeds/pick_apple_messy.txt
    if seed_file.exists():
        return [int(x) for x in seed_file.read_text().split()]
    return list(range(100))
```

已有 seed 文件的 5 个任务：`pick_apple_messy`, `pour`, `multi_grasp`, `place_milk_box`, `cube_in_box`。

每个 seed 文件包含 120 个整数 seed，用空格/换行分隔。评测时按顺序消费，`eval_episodes` 超过种子数时自动截断并警告。

## 7. MultiTaskSimRunner — 多任务评测

```python
class MultiTaskSimRunner:
    """不继承 BaseRunner，而是为每个 task 创建内部 TaskTextSimRunner。
    对外暴露与 BaseRunner 兼容的 run() 接口。"""

    def __init__(self, task_configs, ...):
        for cfg in task_configs:
            task_text = cfg.get("task_text")       # 安全访问，未设置时 fallback 到 task_name
            if task_text is None:
                task_text = cfg["task_name"]
            runner = TaskTextSimRunner(
                task_text=task_text,                # 如 "pick apple"
                task_name=cfg["task_name"],         # 如 "pick_apple_messy"
                env_kwargs=cfg.get("env_kwargs"),
                ...
            )
            self.runners[task_name] = runner

    def run(self, agent, denoise_timesteps, eval_episodes):
        for task_name, runner in self.runners.items():
            try:
                result = runner.run(agent, denoise_timesteps, eval_episodes)
                per_task[task_name] = result
                # 视频以 "task_name_episode_seed" 命名区分任务
            except KeyboardInterrupt:
                raise
            except (RuntimeError, ValueError, AttributeError) as e:
                # 某 task 运行异常 → 标记为失败，继续评测下一个 task
                failed_tasks.append(task_name)
                per_task[task_name] = {"success_rate": None, "avg_steps": None,
                                       "error": str(e), "error_type": type(e).__name__}
            except Exception as e:
                # 未预期的严重错误 → 打印 traceback 后重新抛出
                traceback.print_exc()
                raise

        # 汇总: 各 task success_rate 的未加权平均（失败 task 排除在平均外）
        avg_success_rate = sum(rates) / len(rates) if rates else None
        avg_steps = int(round(sum(steps) / len(steps))) if steps else None
        self.print_summary(per_task, avg_success_rate, avg_steps, rates, failed_tasks)
        return {"success_rate": avg_success_rate, "avg_steps": avg_steps,
                "videos": all_videos, "per_task": per_task, "failed_tasks": failed_tasks}
```

`TaskTextSimRunner` 继承 `SimRunner`，在 `get_action_chunk` 时注入 `task_text` 到 obs dict：

```python
class TaskTextSimRunner(SimRunner):
    def get_action_chunk(self, obs_batch, agent, denoise_timesteps=None):
        obs_batch["task_text"] = [self.task_text]   # 注入语言指令
        return super().get_action_chunk(obs_batch, agent, denoise_timesteps)
```

汇总策略：各 task 的 success_rate 取未加权平均，失败的任务（抛异常）被排除在平均之外。

### 8.2 BaseEnv 生命周期

```python
class BaseEnv(gym.Env):
    def __init__(self, name, frame_skip=20, action_cnt_limit=100, render_mode="human"):
        # 1. 构建 SAPIEN 场景 (PhysX + 光追渲染)
        self.scene, self.viewer = self.build_scene()
        #    - 时间步长 1/200 s
        #    - 光追: 32 spp, 深度 8, OptiX 降噪
        #    - 材质: 静摩擦 1.0, 动摩擦 1.0, 恢复系数 0

        # 2. 加载机器人
        self.robot = XArm7_XHand(scene)

        # 3. 加载物体 (子类实现)
        self.load_objects()

        # 4. 加载传感器
        self.camera = Camera(...)
        self.contact_sensor = ContactNetForce(...)
        self.imagine_pcd_sensor = ImaginePointCloud(...)

        # 5. 初始化状态
        self.initialize_env_state()
        #    action_cnt = 0, video_frames = [], success_condition = False
        #    consistent_success_count = 0

        # 物理时间步
        self.dt = frame_skip * scene.timestep  # 20 * 1/200 = 0.1s
        self.video_fps = int(1 / self.dt)      # 10 fps

    def reset(self, seed, options=None):
        # 1. 机器人复位 (home_qpos 或随机偏移)
        self.robot.reset(random_init=options.get("random_robot"))

        # 2. 物体随机放置 (子类实现)
        self.reset_objects()

        # 3. 物理稳定性检测 (warmup 800 步 + 检测 200 步)
        is_stable, unstable_list = self.check_stable()
        if not is_stable:
            raise RuntimeError(f"Reset Failed for seed {seed}! Unstable: {unstable_list}")

        # 4. 初始化 episode 状态并获取首帧观测
        self.initialize_env_state()
        obs = self.preprocess_obs_data(self.get_sensor_data())
        return obs, info

    def step(self, action):
        self.action_cnt += 1

        # 物理步进: frame_skip 次子步
        for _ in range(self.frame_skip):     # 20 次
            self.robot.balance_passive_force()
            self.robot.apply_action(action)  # PD 目标关节位置
            self.scene.step()                # PhysX 仿真 (1/200 s)

        # 渲染与感知
        self.scene.update_render()
        obs = self.preprocess_obs_data(self.get_sensor_data())
        self.video_frames.append(obs["rgb"])  # 或 viewer_rgb

        # 计算 reward / success
        reward, self.success_condition = self.compute_metric_and_condition()
        done = self.compute_done()            # consistent_success_count >= 3
        truncated = self.compute_truncated()  # action_cnt >= action_cnt_limit

        return obs, reward, done, truncated, info
```

env 生命周期的四个物理阶段：

| 阶段 | 物理步数 | 耗时 | 说明 |
|---|---|---|---|
| **Reset 预热** | 800 | 4.0s | 物体自由沉降，关节被动力衰减 |
| **Reset 稳定性检测** | 200 | 1.0s | 确认所有物体静止 |
| **Step (frame_skip)** | 20 | 0.1s | 施加动作 + 20 次 PhysX 子步 + 渲染 |
| **总计 (100 step episode)** | 100×20+1000 | ~15.0s | 典型 episode 的纯物理仿真时间（不含感知/推理） |

### 8.6 成功判定机制

**两级条件 + 连续确认机制**：

```python
def compute_metric_and_condition(self):
    # 子类实现 → (score, success_condition)
    return 0.0, False

def compute_done(self):
    if self.success_condition:
        self.consistent_success_count += 1
    else:
        self.consistent_success_count = 0
    return self.consistent_success_count >= 3   # 需连续 3 步满足

def compute_truncated(self):
    return self.action_cnt >= self.action_cnt_limit
```

连续 3 步确认防止瞬时满足导致的误判（如物体穿过检测区域后回弹）。每个任务有独立的 `action_cnt_limit`（最大步数上限）。

#### check_stable 物理稳定性检测

环境 reset 后需要确保所有物体在重力下达到静止，否则初始状态的抖动会引入不可控的物理偏差。`check_stable()` 分两个阶段：

```
check_stable():
  # 阶段 1: 预热 (warmup_steps=800)
  for _ in range(800):
      for art in articulation_list:
          qf = art.compute_passive_force()   # 计算关节被动力（重力/弹簧）
          art.set_qf(qf)                      # 施加被动力补偿
      scene.step()                            # 物理步进 (1/200 s)
  # 记录预热后所有 entity 的初始 pose

  # 阶段 2: 稳定性检测 (check_stable_steps=200)
  for _ in range(200):
      # 持续平衡被动力 + 物理步进
      scene.step()
      # 追加每步 pose 到历史列表

  # 判定: 最后 200 步内是否存在显著位移
  for entity in entities:
      final_pose = history[-1]
      for pose in history[-201:]:             # 检测窗口: 最后 201 帧
          rot_diff, pos_diff = get_diff(final_pose, pose)
          if rot_diff > 3.0 or pos_diff > 1.0:  # 旋转 > 3° 或 位移 > 1cm
              unstable_list.append(entity)
              is_stable = False
```

| 阶段 | 步数 | 物理时间 | 目的 |
|---|---|---|---|
| 预热 | 800 | 4.0s | 让物体在重力下自然沉降，关节被动力衰减 |
| 稳定性检测 | 200 | 1.0s | 确认物体已静止，检测窗口内的最大位移/旋转 |
| 总耗时 | 1000 | 5.0s | 每次 reset 的额外物理开销 |

稳定性判断阈值：

| 指标 | 阈值 | 含义 |
|---|---|---|
| 位置偏差 `pos_diff` | > 1.0 cm | 物体在检测窗口内发生了平移 |
| 旋转偏差 `rot_diff` | > 3.0° | 物体在检测窗口内发生了转动 |

`get_diff(pose1, pose2)` 通过四元数内积计算旋转角度差：`rot_diff = 2 * arccos(|q1·q2|) * 180/π`。

不稳定的常见原因：
- 物体堆叠或互相挤压，重力释放后需要更长时间沉降（受限于 `max_attempt_cnt`，拒绝采样已排除部分情况）
- 关节物体（如 OpenBox 的铰链盒盖）的弹簧力与重力耦合，在预热阶段未充分衰减
- 随机种子产生的极端物体位置使接触面过小，微小的物理步进就导致滑动

`check_stable` 失败时抛出 `RuntimeError("Reset Failed for seed {seed}! Unstable objects: {list}")`，`BaseRunner.run()` 捕获后跳过该 seed。

## 10. 常见失败模式分析

评测的目的不仅是获得 success_rate，更重要的是理解失败原因以指导策略改进。按原因可将失败归类为以下几种模式：

| 失败模式 | 典型表现 | 常见原因 | 诊断方式 |
|---|---|---|---|
| **超时未完成** | episode 走满 `action_cnt_limit`，`done=False` | 策略未学会接近目标；动作幅度过小/过于保守 | 对比 `avg_steps` 与 `action_cnt_limit`，若成功的 episode 步数也接近上限则策略偏保守 |
| **抓取失败** | 物体未被稳定抓取，中途掉落 | 灵巧手姿态不准；接触力不足；物体滑脱 | 查看视频中手指与物体的接触情况；MultiGrasp/PickBottle 类任务中集中出现 |
| **碰撞杂物** | 杂物被碰移超过 2.5cm（PickAppleMessy） | 动作轨迹经过杂物区域；灵巧手张开幅度过大 | `check_clutter()` 判定，仅 PickAppleMessy 有此条件 |
| **目标未达** | 物体未被移动到目标区域 | 放置位置不准；接近目标时动作过于激进 | 对比视频中物体最终位置与目标区域的距离 |
| **物体飞溅** | 物体受力过大飞出场景 | PD 控制器刚度 + 策略输出抖动导致瞬时大力 | 视频中物体快速飞出；reset 阶段 unstable 异常增多 |
| **策略输出异常** | 动作值超出关节限位或剧烈震荡 | 正常化逆变换越界；去噪步数不足导致轨迹粗糙 | 对比 denoise_timesteps=5 vs 10 vs 20 的 success_rate 差异 |
| **环境初始化跳过** | seed 被跳过（不计入统计） | 物体随机放置导致无法稳定 | 日志中出现 "env init failed, skipping"；若跳过率 >10% 需调整放置参数 |

### 10.1 如何系统定位失败原因

1. **看视频**：最直接有效的方式。`eval/<timestamp>/denoise_timesteps10/eval_episode_<seed>.mp4`
2. **查 episode_details**：`metrics.json` 中 `episode_details` 列出每个 seed 的 success/steps，找出失败 seed 后重新跑单个 seed 观察
3. **对比 avg_steps**：成功 episode 的平均步数反映策略效率，若接近 `action_cnt_limit` 说明策略虽然成功但不够高效
4. **扫描 denoise_timesteps**：`denoise_timesteps_list=[5,10,20]` 对比可判断推理质量瓶颈——若 5→10 提升显著则推理步数不足；若 10→20 几乎不变则步数已饱和

### 10.2 各任务典型失败分布

| 任务 | 最常见失败模式 | 次要失败模式 | 难度因素 |
|---|---|---|---|
| PickAppleMessy | 碰撞杂物 | 目标未达（苹果未放到盘子上） | 杂物随机 2-4 个，需精细避障 |
| Pour | 目标未达（球未入碗） | 抓取失败（杯子掉落） | 双条件判定（球入碗 + 杯归位） |
| MultiGrasp | 抓取失败 | 超时未完成 | 多物体随机选择，泛化要求高 |
| PlaceMilkBox | 目标未达（盒子未入区） | 超时未完成 | 放置精度要求 |
| OpenBox | 超时未完成 | 抓取失败 | 铰链物体需理解运动学约束 |
| PickBottle | 抓取失败 | 超时未完成 | 简单抓取任务，baseline 对照 |
| StackCups | 目标未达（杯子未叠上） | 超时未完成 | 精确放置 + 层级结构理解 |

### 11.4 多任务结果解读

`MultiTaskSimRunner` 输出 `per_task` 字典，需逐任务检查：

- **task 间 success_rate 差异大**：策略对某些任务 bias 严重，考虑调整 `task_weights`（weighted 采样）或增加弱势任务的训练数据
- **某 task 持续失败**：该任务的 text embedding 可能与训练数据中的文本不一致，检查 task_text 拼写和语义
- **failed_tasks 非空**：该 task 的环境构建或执行阶段抛异常，需单独排查

## 17. 随机性来源与复现性保障

### 17.1 随机性链路

评测中的随机性来自多个层次，逐层分析：

```
eval_sim seed (cfg.eval.seed)
  → (仅影响 Python/pytorch 全局 seed，不影响 episode 随机性)

episode seed (eval_seeds[i])
  ├── env.reset(seed=episode_seed)
  │   ├── np_random 初始化 → 物体位置采样 (sample_centered_positions / sample_uniform_positions)
  │   ├── robot.reset(random_init=True/False) → 机械臂初始姿态偏移
  │   └── check_stable() → 物理确定性（相同 seed + 相同 PhysX 配置 = 相同结果）
  │
  └── agent.predict_action()
      ├── Diffusion (DDPM): x_T ~ N(0, I) — **随机噪声采样**
      │   同一 obs + 同一 seed 下，噪声不同会导致不同动作轨迹
      │
      ├── FlowMatch: x0 ~ randn_like(actions) — **随机初始化**
      │   同一 obs + 同一 seed 下，x0 噪声不同会导致不同 ODE 积分结果
      │
      └── 无显式 seed 控制（agent.predict_action 不接收 seed 参数）
```

### 17.2 复现性分析

| 层面 | 可复现？ | 条件 |
|---|---|---|
| 环境初始化（物体位置） | ✅ 可复现 | 相同 seed + 相同 SAPIEN/PhysX 版本 |
| 物理仿真（step） | ✅ 可复现 | PhysX 确定性模式，相同动作序列 |
| 策略推理（Diffusion） | ❌ 不可复现 | 随机噪声采样无 seed 控制 |
| 策略推理（FlowMatch） | ❌ 不可复现 | x0 随机初始化 |
| episode 级结果（done/steps） | ❌ 不可复现 | 推理随机性 + 物理混沌放大 |

**结论**：同一 seed 多次评测可能得到不同的 success/failure 结果。这是扩散模型 / FlowMatch 推理的固有特性，而非 bug。

### 17.3 提高复现性的实践

1. **增加 `eval_episodes`**：100 episode 的统计均值比 25 episode 更稳定（标准误减少一半）
2. **多轮评测求平均**：对同一 checkpoint 跑 2-3 轮评测，报告均值和标准差
3. **固定推理噪声**（需代码修改）：在 `agent.predict_action()` 中手动 `torch.manual_seed(episode_seed)` 可使相同 seed 产生完全相同的结果
4. **排序对比而非绝对值对比**：跨 checkpoint 的 success_rate 排序通常比绝对值更鲁棒

## 18. FAQ / 常见问题

### Q1: 训练期评测和独立评测的 success_rate 不一致？

训练期评测用 `default_eval_episodes` (25) 个 episode 和 action_decoder 默认 denoise 步数；独立评测用 100 episode 和 `denoise_timesteps_list`。episode 数和 denoise 步数不同会导致统计方差。建议最终报告以独立评测结果为准。

### Q2: 评测时报 "Checkpoint not found: best"？

检查 `<exp_dir>/checkpoints/topk_manifest.json` 是否存在且非空。确保训练时至少完成一次 `evaluate()`（需 `eval_interval_epochs > 0` 且 `env_runner` 已配置），否则不会写入 `test_mean_score`。

### Q3: 环境初始化失败 (Reset Failed / Unstable) 如何处理？

种子文件中的某些 seed 可能产生不稳定的物体配置（如物体堆叠），`BaseRunner` 会自动跳过这些 seed 并打印警告。若大量 seed 被跳过（>10%），检查相应任务的 `sample_centered_positions` / `sample_uniform_positions` 参数。

### Q4: 如何添加新任务的评测？

1. 在 `dexmani_sim/envs/` 创建新任务类继承 `BaseEnv`
2. 在 `dexmani_sim/envs/__init__.py` 导出
3. 在 `mimic_data/eval_seeds/` 添加种子文件（可选，否则使用 range(100)）
4. 训练时 `task_name` 与文件名（不含 .py）一致即可

### Q5: 为什么 done 后环境不自动停止？

`dexmani_sim` 遵循实机 teleoperation 行为：done 后继续执行动作直到 `truncated`（达到步数上限）。评测时 `eval_one_episode` 只在首次 `done=True` 时记录 `task_done_step`，后续 done 被忽略。这避免了机器人"完成即停"与实际部署的 mismatch。

### Q6: 多任务评测的 success_rate 如何汇总？

`MultiTaskSimRunner` 对各 task 的 success_rate 取未加权平均。如果某个 task 因异常失败，该 task 被排除在平均之外（并在输出中标记 `failed_tasks`）。

### Q7: GPU 显存泄漏问题？

SAPIEN 渲染器在长期运行时存在显存累积。`BaseRunner` 每 `clear_cache_freq=25` 个 episode 关闭并重建环境（`env.close()` + `self.make_env()`）。这个频率可在 config 中通过 `env_runner.clear_cache_freq` 调整。

### Q8: denoise_timesteps 对评测结果的影响？

denoise_timesteps 控制去噪/积分的步数，影响动作生成质量和推理速度：
- **Diffusion (DDIM)**：步数越多质量越好但越慢，10 步是配置中的默认平衡点
- **FlowMatch (ODE)**：相对模式下 target_t=dt>0，依赖 consistency 训练提供 target_t 泛化；若不启用 consistency 训练则可能产生分布偏移

### Q9: 视频帧率为什么是 10 fps？

`frame_skip=20`, `dt=1/200s` → `env.dt = 20 * 1/200 = 0.1s` → `video_fps = 1/0.1 = 10`。这是所有任务统一的物理配置。视频录制在 step 而非子步级别，因此每 0.1s 一帧。

### Q10: 评测脚本与训练入口的环境初始化有何不同？

训练时的 `env_runner` 通过 Hydra 实例化但不实际运行（仅在 epoch 末调用 `run()`）。评测脚本重建 agent 和 env_runner 实例，通过 `CheckpointStore` 直接加载 checkpoint（不再经过 workspace），不重建 optimizer/scheduler。

## 附录: 关键约定

1. **独立评测不依赖训练代码**：`eval_sim.py` 只需 `config.yaml` + checkpoint 文件，不需要 trainer 实例。
2. **obs_deque pad 策略与训练一致**：均采用 pad_before 填充最早帧。
3. **EMA 权重是默认评测选择**：`use_ema_for_eval: true`，训练期评测也使用 EMA 模型。
4. **checkpoint "best" 基于 test_mean_score**：该值来自训练期 `eval/success_rate`，需 `eval_interval_epochs > 0` 才会更新。
5. **稳定重置 = 物理预热 800 步 + 检测 200 步**：环境 reset 时的 `check_stable()` 确保物体在重力下静止后才开始 episode。
6. **done 需要连续 3 步满足**：`consistent_success_count >= 3` 防止瞬时波动误判。
7. **种子文件在外部包**：`dexmani_sim/mimic_data/eval_seeds/`，由 `SimRunner.get_seed_list()` 读取。
8. **环境使用 importlib 动态导入**：task_name 需同时匹配文件名（snake_case）和类名（PascalCase）。
9. **视频来自 step 级帧累积**：`env.get_video()` 返回整个 episode 的 `np.stack(video_frames)` 并清空内部缓存。
10. **MultiTaskSimRunner 不继承 BaseRunner**：采用组合模式，内部持有多个 `TaskTextSimRunner` 实例，对外暴露兼容接口。
11. **render_mode 在评测时为 "rgb_array"**：无 viewer 窗口，避免 GUI 依赖；视频帧来自相机 rgb 输出。
12. **action_dim = 19 = 7 (arm) + 12 (hand)**：评测输出的 control_action 直接作为 PD 控制器目标关节位置。
